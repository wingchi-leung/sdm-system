# 鉴权说明：登录用哪张表？

## 两张表各干什么

| 表名 | 用途 | 是否有密码 |
|------|------|------------|
| **user** | 存「报名用户」信息、黑名单（isblock、block_reason） | **没有**密码字段，不用于登录 |
| **admin_user** | 存「管理员」登录账号（用户名 + 密码哈希） | **有** password_hash，**登录鉴权用这张表** |

- **登录**：前端传 `username`、`password` → 后端在 **admin_user** 里按 `username` 查一条，用 **password_hash** 校验密码 → 通过则发 JWT。
- **user 表**：只存参与活动的人（姓名、证件号、手机等）和是否拉黑，不参与登录；以后若要「管理员同时对应一个 user」，可在 admin_user 里填 `user_id` 关联 user。

所以：**鉴权只看 admin_user，不看 user 表。**

---

## 你设了密码 mysql123456 但登录报「用户名或密码错误」时

### 先做一步：本地校验库里 hash 是否和密码一致

在 **backend** 目录执行（把密码换成你实际要登录的）：

```bash
python scripts/verify_admin_password.py admin mysql123456
```

- 若输出 **「校验通过」**：说明库里 password_hash 与密码一致，问题在 Flutter 或网络（见下）。
- 若输出 **「校验失败」**：说明库里 hash 与密码不一致，按脚本里的提示重新生成 hash 并 `UPDATE admin_user SET password_hash='...' WHERE username='admin';`。

### 再按下面逐项检查

1. **密码拼写是否完全一致**
   - 你打的是 **mysql123456** 还是 **msyql123456**（少一个 l）？必须和「生成 hash 时用的」以及「库里 hash 对应的」完全一致。

2. **确认用的是 admin_user 表**
   - 密码必须写在 **admin_user** 的 **password_hash** 里，不是 user 表。

3. **确认 password_hash 是用「你要登录的密码」生成的**
   - 密码是 **mysql123456** 时，必须用：`python scripts/hash_admin_password.py mysql123456` 生成 hash，整段复制进库里。

4. **确认登录时用户名一致**
   - 库里是 `username='admin'` 时，前端用 **admin** 登录，区分大小写。

## 首次：建 admin_user 并插入管理员（密码 mysql123456）

在项目 **backend** 目录下：

```bash
# 1. 生成密码 mysql123456 的 hash
python scripts/hash_admin_password.py mysql123456
# 会输出一长串，例如：$2b$12$xxxx...

# 2. 在 MySQL 里建表（若还没建过）
# 执行 migrations/create_admin_user.sql 里的 CREATE TABLE

# 3. 插入管理员（把下面 <hash> 换成第 1 步的整段输出）
# INSERT INTO admin_user (username, password_hash) VALUES ('admin', '<hash>');
```

例如第 1 步输出是 `$2b$12$abc...xyz`，则：

```sql
INSERT INTO admin_user (username, password_hash) VALUES ('admin', '$2b$12$abc...xyz');
```

之后用 **用户名 admin**、**密码 mysql123456** 登录即可。
