# 小程序安全检查清单

> 更新日期：2026-05-08
> 适用范围：`miniprogram/`、与小程序直接交互的后端接口

## 账号与页面状态

- 敏感页面必须在 `onLoad` 和 `onShow` 都校验当前身份，不能只在首次进入时校验。
- 管理员页面在身份失效时必须立即清空报名人员、签到、统计、用户列表、活动表单等敏感数据。
- 普通用户页面在身份失效或切换为管理员时必须清空“我的报名”“我的订单”等用户侧数据。
- 页面上按钮隐藏只作为体验控制，后端接口必须独立做权限和租户校验。
- 账号切换、退出登录、租户切换时，不允许复用上一账号的 `require_bind_info`、手机号、头像、订单摘要等临时数据。

## 本地缓存

- 用户相关缓存必须至少按 `tenantCode + userId` 隔离；不能只按租户存储。
- 支付订单摘要、待支付订单、本地订单历史必须按当前用户隔离。
- Storage 读取失败只能影响本地展示，不能影响后端支付、报名、权限判断的最终结果。
- 不要把后端可重新获取的敏感列表长期缓存在本地。

## 报名与支付

- 小程序报名和支付 payload 不提交 `user_id`，用户身份必须由后端根据 token 推导。
- 支付前必须确认后端返回了 `order_no` 和完整 `payment_params`，否则不能调用 `wx.requestPayment`。
- 查询支付订单时必须对订单号做 URL 编码。
- 支付金额、订单所属用户、活动是否可报名、是否重复报名，最终都以后端校验为准。
- 小程序只能做体验层校验，不能把前端校验当作安全边界。

## 后端接口

- 支付、报名、用户管理、活动社区接口必须使用 token 中的用户、租户和角色上下文。
- 普通用户不能调用管理员报名、签到、统计、用户管理接口。
- 管理员不能直接报名、支付报名或发表评论。
- 活动管理员只能管理自己授权活动类型下的活动。
- 社区文章和评论读取必须限制为管理员或已报名用户。
- 订单查询必须校验订单所属租户和所属用户。

## 手工验证路径

1. 管理员登录，进入“报名人员”“签到记录”“报名统计”“用户管理”“编辑活动”“发布动态”等页面。
2. 不关闭小程序，退出管理员账号，切换普通用户登录。
3. 从页面栈返回上述页面，确认旧报名人员、签到、统计、用户列表、活动表单不会继续展示。
4. 普通用户登录，进入“我的报名”“我的订单”“活动动态详情”。
5. 退出普通用户并切换管理员账号，返回上述页面，确认旧订单和报名记录不会继续展示，评论入口不可用。
6. 对付费活动发起报名，分别验证正常支付、取消支付、创建订单失败、支付参数缺失、订单恢复查询。
7. 使用另一个普通用户登录，确认不能看到前一个用户的本地订单历史。

- `miniprogram/tests/mine-data.test.js`：订单缓存按租户和用户隔离。
