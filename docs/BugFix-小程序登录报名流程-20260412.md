# Bug修复记录：小程序登录与报名流程

**日期**：2026-04-12  
**涉及模块**：小程序（miniprogram）、后端（backend）

## 1. 后端启动报错：payment_order 表缺少字段

**现象**：关闭过期订单定时任务报错 `Unknown column 'payment_order.participant_snapshot'`

**原因**：`backend/app/schemas.py` 中 `PaymentOrder` ORM 模型已添加 `participant_snapshot` 字段，但数据库表未执行对应的 ALTER 语句。

**修复**：在 `backend/field.sql` 追加：
```sql
ALTER TABLE payment_order ADD COLUMN participant_snapshot TEXT NULL COMMENT '报名信息快照(JSON)' AFTER phone;
```

## 2. 绑定信息页找不到证件号码输入框

**现象**：登录后进入「完善信息」页，证件类型已默认选中「大陆身份证」，但证件号码输入框不显示。

**原因**：`bind-user-info.wxml` 中证件号输入框的显示条件为 `wx:if="{{identityTypeIndex > 0}}"` ，而 JS 初始值 `identityTypeIndex = 0`，导致 `0 > 0` 为 false，输入框不渲染。必须重新点一次 picker 才会出现。

**修复**：
- `identityTypeIndex` 初始值改为 `-1`（表示未选择）
- `identity_type` 初始值改为 `''`
- 显示条件改为 `identityTypeIndex >= 0`
- picker 占位文字改用三元表达式 `identityTypeIndex >= 0 ? ... : '请选择证件类型'`

## 3. 手机号校验报错（真实手机号无法通过）

**现象**：用微信授权手机号登录后，绑定页回显的手机号提示「请填写正确的手机号」。

**原因**：后端 `_get_phone_number_from_wechat` 函数优先取微信返回的 `phoneNumber` 字段，该字段格式为 `+8613812345678`（含国家区号），前端正则 `^1[3-9]\d{9}$` 无法匹配。

**修复**（`backend/app/api/v1/endpoints/auth.py`）：
```python
# 优先取 purePhoneNumber（纯11位），phoneNumber 含 +86 前缀
phone = phone_info.get("purePhoneNumber") or phone_info.get("phoneNumber", "")
if phone.startswith("+86"):
    phone = phone[3:]
```

**同时修复**：小程序手机号输入框 `type="number"` 改为 `type="digit"`，并在 `onPhoneInput` 里用 `.replace(/\D/g, '')` 过滤非数字字符，彻底避免部分机型允许输入非数字的问题。

## 4. 绑定页点提交没反应

**现象**：填写完所有信息后点「提交」按钮，页面毫无反应。

**原因**：两个 bug 叠加：
1. `validateForm` 在身份证格式校验出错时执行 `this.setData({ identityError })` 然后 `return null`，`submit` 收到 `null` 误判为"无错误"，但 `setData` 异步，紧接着读 `this.data.identityError` 为旧值，导致逻辑混乱。
2. `identityError` 从未在 wxml 中渲染，用户看不到任何错误提示。

**修复**：
- `validateForm` 中身份证出错统一 `return identityError`，不再单独 setData
- wxml 错误框改为 `wx:if="{{error || identityError}}"` 同时显示两种错误

- 手机号输入框同样改为 `type="digit"` + 过滤
- `sexOptions` 删除「其他」选项，只保留「男/女」

## 6. 管理员身份登录看不到报名按钮

**现象**：管理员登录后，活动详情页没有「立即报名」按钮。

**原因**：报名按钮区域被 `wx:if="{{!isAdmin}}"` 包裹，管理员身份时整块不渲染。

**修复**：去掉 `!isAdmin` 限制，报名按钮对所有人可见，管理功能面板单独存在。

## 7. 报名页点提交没反应（用户资料异步加载问题）

**现象**：进入报名页后立即点提交，没有任何反应。

**原因**：
1. `onLoad` 中调用 `ensureLoggedIn` 但未 `return` 拦截，未登录时也会继续加载页面
2. `loadUserProfile()` 是异步的，若用户进页面后立即点提交，`name`/`phone` 尚为空字符串，`validateForm` 返回 false，但错误提示在页面底部用户看不到，感觉像「没反应」

**修复**（`register.js`）：
- `onLoad` 直接做登录判断并 `return` 拦截，不再依赖 `ensureLoggedIn`
- `validateForm` 增加 `loading` 状态判断，给出「页面加载中，请稍候再试」的明确提示
- `submit` 验证失败后调用 `wx.pageScrollTo({ selector: '.error-box' })` 自动滚动到错误提示

## 8. 点击「立即报名」按钮完全没反应（根本原因）

**现象**：点击活动详情页的「立即报名」按钮，页面毫无变化。

**原因**：**`pages/register/register` 未在 `app.json` 的 `pages` 列表中注册。** 微信小程序所有页面必须在 `app.json` 声明，跳转未注册页面时 `wx.navigateTo` 静默失败，没有任何报错或提示。

**修复**（`app.json`）：
```json
"pages/register/register"
```
加入 pages 列表。

## 9. 小程序登录报错：wx_openid 唯一约束冲突导致 Session 卡死

**现象**：手机号登录时后端报 `sqlalchemy.exc.PendingRollbackError`，原始异常为 `(1062, "Duplicate entry '1-oWKPZ5G...' for key 'user.uk_tenant_wx_openid'")`。

**原因**：两个 bug 叠加：
1. `phone_login` 绑定 openid 前只检查了当前用户是否已有 openid（`not user.wx_openid`），但未检查该 openid 是否已被同租户其他用户占用，直接 UPDATE 触发唯一约束冲突。
2. `except Exception: pass` 吞掉了异常但没有调用 `db.rollback()`，session 进入 `PendingRollbackError` 状态，后续所有数据库操作全部报错。

**修复**（`backend/app/api/v1/endpoints/auth.py`，`phone_login` 函数）：
```python
existing = crud_user.get_user_by_wx_openid(db, openid, tenant.id)
if existing and existing.id != user.id:
    existing.wx_openid = None   # 先解绑旧用户（手机号登录优先级更高）
user.wx_openid = openid
db.commit()
# except 块加上：
db.rollback()
```

## 10. 报名支付报错：WeChatPay 初始化参数名错误

**现象**：创建支付订单时报 `TypeError: WeChatPay.__init__() got an unexpected keyword argument 'merchant_certid'`。

**原因**：`wechatpayv3` 库升级到 v2.x 后将初始化参数 `merchant_certid` 重命名为 `cert_serial_no`，代码仍使用旧参数名。

**修复**（`backend/app/services/wechat_pay.py`）：
```python
# 旧
merchant_certid=settings.WECHAT_PAY_SERIAL_NO
# 改为
cert_serial_no=settings.WECHAT_PAY_SERIAL_NO
```

## 11. 报名支付报错：No wechatpay platform certificate

**现象**：修复参数名后，初始化仍失败，报 `No wechatpay platform certificate, please double check your init params`。

**原因（分两层）**：
1. **第一层**：`WECHAT_PAY_SERIAL_NO` 配置的是错误的序列号（`6516E892...`），与实际证书文件 `apiclient_cert.pem` 中的序列号（`7C99C423...`）不一致，导致签名验证失败返回 401。通过 Python 读取证书文件确认：
   ```python
   from cryptography import x509
   cert = x509.load_pem_x509_certificate(open('apiclient_cert.pem','rb').read())
   print(format(cert.serial_number, 'X'))  # 7C99C423F531D4AB49D64ADF9FA9250DAFE16986
   ```
2. **第二层**：更新 SERIAL_NO 后仍报错。微信 API 返回 `404 RESOURCE_NOT_EXISTS`，错误信息为「无可用的平台证书，请在商户平台-API安全申请使用微信支付公钥」。该商户号为新版商户，默认走**微信支付公钥模式**，不提供平台证书接口（`/v3/certificates`），而代码只实现了旧的平台证书模式。

**修复**：
- `.env` 更新 `WECHAT_PAY_SERIAL_NO=7C99C423F531D4AB49D64ADF9FA9250DAFE16986`
- 商户平台下载微信支付公钥，配置新增环境变量：
  ```env
  WECHAT_PAY_PUBLIC_KEY_PATH=C:\Users\g0132\.wechat_pay\wx_public_key.pem
  WECHAT_PAY_PUBLIC_KEY_ID=PUB_KEY_ID_...
  ```
- 完整重构 `wechat_pay.py`（见下条）

## 12. 微信支付服务重构：统一支持公钥/平台证书双模式

**背景**：原代码将两种验签模式混写在 `__init__` 中，`cert_dir` 在公钥模式下被多余创建，类单例与模块全局变量双重冗余，初始化错误信息不明确。

**重构内容**（`backend/app/services/wechat_pay.py`）：

| 问题 | 修复方式 |
|------|----------|
| 两种模式逻辑混写 | 提取 `_build_wxpay_client()` 纯函数，按配置分支清晰处理 |
| 公钥模式下多余创建 `cert_dir` | 公钥模式传 `cert_dir=None`，证书模式才创建目录 |
| 类单例 + 模块全局变量双重冗余 | 去掉 `__new__` 类单例，改用 `@cached_property` 懒加载 `_client`，模块级 `_service` 保持单例 |
| 初始化失败信息不明确 | `try/except` 包裹初始化，区分模式输出诊断信息 |

**模式自动判断规则**：
- 同时配置了 `WECHAT_PAY_PUBLIC_KEY_PATH` 和 `WECHAT_PAY_PUBLIC_KEY_ID` → 公钥模式（新商户）
- 未配置上述两项 → 平台证书模式（旧商户，库自动拉取缓存）
