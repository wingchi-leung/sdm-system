# Bug修复记录：小程序登录与报名流程

**日期**：2026-04-12  
**涉及模块**：小程序（miniprogram）、后端（backend）

---

## 1. 后端启动报错：payment_order 表缺少字段

**现象**：关闭过期订单定时任务报错 `Unknown column 'payment_order.participant_snapshot'`

**原因**：`backend/app/schemas.py` 中 `PaymentOrder` ORM 模型已添加 `participant_snapshot` 字段，但数据库表未执行对应的 ALTER 语句。

**修复**：在 `backend/field.sql` 追加：
```sql
ALTER TABLE payment_order ADD COLUMN participant_snapshot TEXT NULL COMMENT '报名信息快照(JSON)' AFTER phone;
```

---

## 2. 绑定信息页找不到证件号码输入框

**现象**：登录后进入「完善信息」页，证件类型已默认选中「大陆身份证」，但证件号码输入框不显示。

**原因**：`bind-user-info.wxml` 中证件号输入框的显示条件为 `wx:if="{{identityTypeIndex > 0}}"` ，而 JS 初始值 `identityTypeIndex = 0`，导致 `0 > 0` 为 false，输入框不渲染。必须重新点一次 picker 才会出现。

**修复**：
- `identityTypeIndex` 初始值改为 `-1`（表示未选择）
- `identity_type` 初始值改为 `''`
- 显示条件改为 `identityTypeIndex >= 0`
- picker 占位文字改用三元表达式 `identityTypeIndex >= 0 ? ... : '请选择证件类型'`

---

## 3. 手机号校验报错（真实手机号无法通过）

**现象**：用微信授权手机号登录后，绑定页回显的手机号提示「请填写正确的手机号」。

**原因**：后端 `_get_phone_number_from_wechat` 函数优先取微信返回的 `phoneNumber` 字段，该字段格式为 `+8613812345678`（含国家区号），前端正则 `^1[3-9]\d{9}$` 无法匹配。

**修复**（`backend/app/api/v1/endpoints/auth.py`）：
```python
# 优先取 purePhoneNumber（纯11位），phoneNumber 含 +86 前缀
phone = phone_info.get("purePhoneNumber") or phone_info.get("phoneNumber", "")
if phone.startswith("+86"):
    phone = phone[3:]
```

**同时修复**：小程序手机号输入框 `type="number"` 改为 `type="digit"`，并在 `onPhoneInput` 里用 `.replace(/\D/g, '')` 过滤非数字字符，彻底避免部分机型允许输入非数字的问题。

---

## 4. 绑定页点提交没反应

**现象**：填写完所有信息后点「提交」按钮，页面毫无反应。

**原因**：两个 bug 叠加：
1. `validateForm` 在身份证格式校验出错时执行 `this.setData({ identityError })` 然后 `return null`，`submit` 收到 `null` 误判为"无错误"，但 `setData` 异步，紧接着读 `this.data.identityError` 为旧值，导致逻辑混乱。
2. `identityError` 从未在 wxml 中渲染，用户看不到任何错误提示。

**修复**：
- `validateForm` 中身份证出错统一 `return identityError`，不再单独 setData
- wxml 错误框改为 `wx:if="{{error || identityError}}"` 同时显示两种错误

---

## 5. 绑定页年龄可以填文字、性别有「其他」选项

**现象**：年龄输入框可以输入任意字符；性别选项有「男/女/其他」。

**修复**：
- 年龄输入框 `type` 改为 `digit`，`onAgeInput` 里用 `.replace(/\D/g, '')` 过滤
- 手机号输入框同样改为 `type="digit"` + 过滤
- `sexOptions` 删除「其他」选项，只保留「男/女」

---

## 6. 管理员身份登录看不到报名按钮

**现象**：管理员登录后，活动详情页没有「立即报名」按钮。

**原因**：报名按钮区域被 `wx:if="{{!isAdmin}}"` 包裹，管理员身份时整块不渲染。

**修复**：去掉 `!isAdmin` 限制，报名按钮对所有人可见，管理功能面板单独存在。

---

## 7. 报名页点提交没反应（用户资料异步加载问题）

**现象**：进入报名页后立即点提交，没有任何反应。

**原因**：
1. `onLoad` 中调用 `ensureLoggedIn` 但未 `return` 拦截，未登录时也会继续加载页面
2. `loadUserProfile()` 是异步的，若用户进页面后立即点提交，`name`/`phone` 尚为空字符串，`validateForm` 返回 false，但错误提示在页面底部用户看不到，感觉像「没反应」

**修复**（`register.js`）：
- `onLoad` 直接做登录判断并 `return` 拦截，不再依赖 `ensureLoggedIn`
- `validateForm` 增加 `loading` 状态判断，给出「页面加载中，请稍候再试」的明确提示
- `submit` 验证失败后调用 `wx.pageScrollTo({ selector: '.error-box' })` 自动滚动到错误提示

---

## 8. 点击「立即报名」按钮完全没反应（根本原因）

**现象**：点击活动详情页的「立即报名」按钮，页面毫无变化。

**原因**：**`pages/register/register` 未在 `app.json` 的 `pages` 列表中注册。** 微信小程序所有页面必须在 `app.json` 声明，跳转未注册页面时 `wx.navigateTo` 静默失败，没有任何报错或提示。

**修复**（`app.json`）：
```json
"pages/register/register"
```
加入 pages 列表。
