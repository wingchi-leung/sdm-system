# SDM 后端自动化测试方案

## 一、测试基础设施

### 1.1 测试数据库

**使用 SQLite 内存数据库**

```python
# tests/conftest.py
SQLALCHEMY_DATABASE_URL = "sqlite:///:memory:"

engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False},
    echo=False,
)
```

**特点：**
- **内存存储**：测试数据仅存在于内存中，测试结束后自动清除
- **高速执行**：无磁盘 I/O，测试运行速度快
- **完全隔离**：每个测试用例独立运行，互不影响
- **自动创建表结构**：每个测试函数开始时自动创建所有表

### 1.2 数据回滚机制

**每个测试函数完全隔离，数据自动回滚**

```python
@pytest.fixture(scope="function")
def db_session() -> Generator[Session, None, None]:
    """创建测试数据库会话，每个测试函数独立使用"""
    Base.metadata.create_all(bind=engine)  # 创建所有表

    connection = engine.connect()
    transaction = connection.begin()  # 开始事务

    session = TestingSessionLocal(bind=connection)

    # 嵌套事务支持
    session.begin_nested()

    yield session  # 提供给测试使用

    # 清理阶段
    session.close()
    transaction.rollback()  # 回滚所有更改
    connection.close()
    Base.metadata.drop_all(bind=engine)  # 删除所有表
```

**回滚机制说明：**

1. **测试级别隔离**：每个测试函数（`@pytest.fixture(scope="function")`）获得独立的数据库会话
2. **嵌套事务**：使用 `begin_nested()` 创建 savepoint，支持测试内的事务控制
3. **自动回滚**：测试结束后，所有更改通过 `transaction.rollback()` 回滚
4. **表结构重建**：下次测试时重新创建表结构

**示例：**
```python
def test_create_user(db_session):
    # 创建用户
    user = User(name="测试", phone="13800138000")
    db_session.add(user)
    db_session.commit()

    # 测试断言
    assert user.id is not None

# 测试结束后，user 记录自动回滚，数据库为空
```

## 六、运行测试

### 6.1 基本命令

```bash
# 进入后端目录
cd backend

# 运行所有测试
pytest

# 运行特定类型测试
pytest -m unit          # 单元测试
pytest -m api           # API 测试
pytest -m integration   # 集成测试
pytest -m e2e           # E2E 测试

# 运行特定文件
pytest tests/unit/test_security.py
pytest tests/api/test_auth.py

# 运行特定测试类
pytest tests/unit/test_security.py::TestPasswordSecurity

# 运行特定测试用例
pytest tests/unit/test_security.py::TestPasswordSecurity::test_hash_password_success
```

### 6.2 高级选项

```bash
# 详细输出
pytest -vv

# 显示打印内容
pytest -s

# 短格式错误信息
pytest --tb=short

# 只显示失败测试摘要
pytest --tb=line

# 并行运行（加速）
pytest -n auto

# 生成 HTML 覆盖率报告
pytest --cov=app --cov-report=html

# 生成终端覆盖率报告
pytest --cov=app --cov-report=term-missing

# 组合使用
pytest -v -n auto --cov=app --cov-report=html
```

### 6.3 调试选项

```bash
# 遇到错误时进入调试器
pytest --pdb

# 第一个失败时停止
pytest -x

# 遇到错误时停止
pytest --maxfail=3

# 只运行上次失败的测试
pytest --lf

# 重新运行所有测试（忽略缓存）
pytest --ff
```

## 八、测试数据工厂

### 8.1 Factory Boy 工厂

```python
# tests/factories.py

class UserFactory(SQLAlchemyModelFactory):
    """用户工厂"""
    class Meta:
        model = User
        sqlalchemy_session_persistence = "commit"

    name = factory.LazyAttribute(lambda _: fake.name())
    phone = factory.Sequence(lambda n: f"138{n:08d}")
    identity_number = factory.LazyAttribute(lambda _: fake.ssn()[:18])
    password_hash = factory.LazyAttribute(lambda _: hash_password("password123"))

class ActivityFactory(SQLAlchemyModelFactory):
    """活动工厂"""
    class Meta:
        model = Activity
        sqlalchemy_session_persistence = "commit"

    activity_name = factory.LazyAttribute(lambda _: f"测试活动_{fake.word()}")
    start_time = factory.LazyAttribute(lambda _: datetime.now())
    status = 1
```

# 实际实现
def get_user(db: Session, user_id: int, tenant_id: int) -> User | None:
    return db.query(User).filter(
        User.id == user_id,
        User.tenant_id == tenant_id
    ).first()

    user = get_user(db_session, user_id=1, tenant_id=1)

## 十三、常见问题

### Q1: 测试数据会污染生产数据库吗？

**不会**。测试使用独立的 SQLite 内存数据库，与生产环境完全隔离。

### Q2: 测试运行很慢怎么办？

1. 使用并行测试：`pytest -n auto`
2. 只运行特定测试：`pytest -m unit`
3. 使用 `--lf` 只运行失败的测试

### Q3: 如何调试失败的测试？

```bash
# 打印详细输出
pytest -vv -s tests/unit/test_security.py::test_name

# 使用 pdb 调试
pytest --pdb tests/unit/test_security.py::test_name

# 只运行失败的测试
pytest --lf
```

### Q4: 如何 Mock 外部服务？

```python
def test_with_mock(client, mocker):
    # Mock 微信 API
    mock_response = {"openid": "test_openid"}
    mocker.patch("httpx.AsyncClient.get", return_value=mock_response)

    response = client.post("/api/v1/auth/wechat-login", ...)
    assert response.status_code == 200
```
