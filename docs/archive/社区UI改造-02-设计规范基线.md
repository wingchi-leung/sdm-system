## 报告说明

本文件是「小程序社区模块 UI 大改」项目第一阶段的设计规范基线,目标是:

- **摸清当前设计令牌** —— 梳理 `app.wxss` 与 6 个社区页面 wxss 里的所有 token,做健康度评估;
- **暴露令牌缺口与冲突** —— 把摸底报告 §5 的现象量化为具体 token 列表;
- **准备组件规范模板** —— 按原子设计法为社区模块列组件清单 + 空白模板,**等设计稿到位后由设计师 agent 填实**;
- **给出设计稿落地流程** —— 一份给设计师 agent 自己(Phase 2 时)用的工作流清单;
- **给前端 agent 一份视觉协作清单** —— 让 Phase 2 实施期高效对接。

**重要前提**:
- 用户会自己出设计稿。本文档在设计稿未到位前**不写死视觉值**,只给**骨架**与**字段**。
- 等设计稿到位后,设计师 agent 会把本文档的"空白规范模板"逐一填实,再交付前端 agent。
- 一旦填实完成,本文档升级为 v2「设计规范正式版」,与 `app.wxss` 同步生效。

### 1.9 字号/字重/动效 —— 小结

- **色彩系统**最不健康(❌ 多),核心痛点:
  - `--green` / `--success` / `--accent` 等"语义名"被覆写成黑色
  - 多个硬编码色值(`#111827` / `#12b76a` / `#e5484d` / 多个 rgba 阴影)未 token 化
  - 审核页与社区其他页色彩风格割裂
- **字号/字重/行高**基本可用,只需 token 化收口;
- **圆角**有个小坑: `--radius-md` 与 `--radius-lg` 值完全相同;
- **阴影**有 4 种硬编码 hero/卡片/CTA 阴影,**未走 token**;
- **动效**过渡时长不统一(0.3/0.4/0.5s 混用)。

## 3. 组件设计规范模板

> **分级方法**: 原子层(不可再分) → 分子层(由原子组合) → 组织层(由分子组合,带业务) → 页面模板层(布局)。
> **本节是空白模板**,等设计稿到位后由设计师 agent 把"待设计稿填实"字段一一填充为实际值。

### 3.1 原子层 (Atoms)

#### Avatar(头像)

- **用途**: 用户头像 / 频道头像
- **关键属性**:
  - `size`: `sm: 24rpx` / `md: 40rpx` / `lg: 72rpx`
  - `shape`: `circle` / `square`(带圆角)
  - `src`: 头像 URL
  - `fallback`: 首字母 / 默认图
  - `badge`: 可选,右上角小红点
- **视觉状态**:
  - 默认: `[待设计稿填实]`
  - 按下: `[待设计稿填实]`
  - 加载中: 骨架 / 灰底
  - 错误: fallback
- **令牌依赖**: `var(--radius-full)` 或 `var(--radius-md)`、`var(--background-subtle)`(fallback 底)
- **[待设计稿填实]**: 3 尺寸、边框、placeholder 渐变、点击反馈

#### Badge(角标 / 未读小红点)

- **用途**: 未读数提示
- **关键属性**:
  - `count`: 数字(>99 时显示 "99+")
  - `dot-only`: 布尔(只显示点)
  - `position`: `top-right` / `inline`
  - `variant`: `default` / `primary` / `danger`
- **视觉状态**:
  - 默认: `[待设计稿填实]`
  - count = 0: 不渲染
- **令牌依赖**: `var(--ink)` 或 `var(--error)`(背景)、`#FFFFFF`(文字)
- **[待设计稿填实]**: 尺寸、内边距、字号、变体色

#### Tag(标签)

- **用途**: "官方" / "置顶" / 状态标识
- **关键属性**:
  - `text`: 标签文字
  - `variant`: `default` / `official` / `pinned` / `success` / `warning` / `danger`
  - `size`: `sm: 22rpx` / `md: 24rpx`
- **视觉状态**:
  - 默认: `[待设计稿填实]`
- **令牌依赖**: `var(--success-bg)` / `var(--success)`、`var(--warning-bg)` / `var(--warning)`、`var(--error-bg)` / `var(--error)`
- **[待设计稿填实]**: 各 variant 颜色、圆角、字重

#### Icon(图标)

- **用途**: 通用图标
- **关键属性**:
  - `name`: 字符串(对应 svg / iconfont 名字)
  - `size`: `16/20/24/32/40 rpx`
  - `color`: 文字色 / 主题色
- **视觉状态**:
  - 默认: `[待设计稿填实]`
  - 按下: 透明度 0.7
- **令牌依赖**: `var(--text-primary)` / `var(--text-secondary)` / `var(--text-tertiary)`
- **[待设计稿填实]**: 图标清单(社区模块至少要: 首页/通知/发布/编辑/删除/更多/置顶/官方/评论/点赞/分享/收藏/展开/收起/搜索/添加/关闭/箭头/空状态插画)
- **图标方案**:**优先 svg 矢量**(小程序支持),fallback 走 iconfont / 单色 png 2x/3x

#### Button(按钮)

- **用途**: 主 CTA / 次 CTA / 文字按钮
- **关键属性**:
  - `variant`: `primary` / `secondary` / `text` / `danger` / `ghost`
  - `size`: `sm: 64rpx` / `md: 80rpx` / `lg: 88rpx`
  - `block`: 布尔(通栏)
  - `loading`: 布尔
  - `disabled`: 布尔
  - `icon`: 可选图标
- **视觉状态**:
  - 默认: `[待设计稿填实]`
  - 按下: `[待设计稿填实]`
  - 禁用: 透明度 0.4
  - 加载: 显示 spinner,文字隐藏
- **令牌依赖**: `var(--ink)` / `#FFFFFF` / `var(--border)` / `var(--radius-md)`
- **[待设计稿填实]**: 5 个 variant 的颜色 + 阴影

- **用途**: 创建/编辑频道时选择头像
- **关键属性**:
  - `value`: 当前头像 URL
  - `onChange(url)`: 选中回调
  - `max-size`: 2MB
  - `shape`: `circle` / `square`
- **视觉状态**:
  - 未选择: `[待设计稿填实]` 占位插画 / "+" 按钮
  - 已选择: 显示头像 + 浮层"更换/删除"
  - 上传中: 进度遮罩
  - 错误: 重试按钮
- **令牌依赖**:
  - 占位: `var(--background-subtle)` + `var(--text-tertiary)` 图标
  - 边框: `var(--form-border)` 虚线
- **[待设计稿填实]**:
  - 头像形状与默认尺寸(频道头像推荐 96rpx / 128rpx)
  - "+" 图标与文字
  - 选中后的覆盖层样式

#### ChannelCard(频道列表项)

- **用途**: 社区首页的频道列表项
- **关键属性**:
  - `channel`: 频道数据(名称、描述、头像、成员数、admin 标志)
  - `onTap`
  - `onLongPress`(管理员可编辑)
- **视觉状态**:
  - 默认: `[待设计稿填实]`
  - 按下: 透明度 0.86 / 1rpx 位移
- **令牌依赖**:
  - 容器: `var(--background-card)` + `var(--shadow-card)` + `var(--radius-xl)`
  - 头像: Avatar
  - 文字: `var(--ink)` / `var(--text-tertiary)`
- **[待设计稿填实]**:
  - 头像尺寸、名称字号、描述行数
  - admin 角标(可选,不做强约束)

#### NotificationItem(站内信项)

- **用途**: 站内信列表的每条
- **关键属性**:
  - `notification`: 通知对象
  - `unread`: 布尔
  - `onTap`: 点击进入详情
  - `onAccept` / `onReject`(邀请类)
  - `onMarkRead`
- **视觉状态**:
  - 未读: 左侧小圆点 + 边框加深
  - 已读: 普通样式
  - 按下: 1rpx 位移
- **令牌依赖**:
  - 未读 dot: `var(--ink)`(或 `var(--error)`)
  - type-pill: `var(--background-subtle)` + `var(--text-secondary)`
  - state-pill: `var(--success-bg)` / `var(--warning-bg)` / `var(--reject-bg)`
- **[待设计稿填实]**:
  - 头像 / icon 是否在左侧
  - 时间显示位置
  - 接受/拒绝按钮形态(行内 / 底部)

### 3.4 页面模板层 (Templates)

#### FeedPage(瀑布流流式页面)

- **布局**: `page > header(hero/eyebrow + 标题 + 工具栏) → tabs(频道切换,可选) → masonry(WaterfallCard × N,2 列) → loading/end`
- **关键约束**:
  - 2 列等宽,列间距 `[待设计稿填实]`
  - 滚动到底自动加载更多
  - 顶部 sticky tabs
- **令牌依赖**:
  - 页面底: `var(--surface-gradient-hero)`(顶 36% 渐变到底白)
  - 列间距: `var(--card-gap)`
  - 卡片: 复用 `WaterfallCard`

#### DetailPage(若保留)

- **布局**: `page > post-card(完整正文) → comment-section(评论流) → comment-bar(底部固定)`
- **角色**: 仅作为分享落点 / 系统消息深链入口,主流交互不再依赖
- **令牌依赖**: 复用 `community-post-detail` 现有 token

#### EditorPage(发布/编辑页)

- **布局**: `page > form(频道选择 + RichTextEditor + 字数计数 + 错误提示) → btn-submit(底部固定)`
- **关键约束**:
  - 工具栏在编辑器顶部或底部固定
  - 字数计数右下角
  - 提交按钮底部固定,滚动时仍可见
- **令牌依赖**:
  - 表单: `var(--form-surface)` + `var(--form-border)`
  - 提交: 主按钮规格

## 5. 设计稿落地流程(关键产出)

> 用户给出设计稿后,设计师 agent 的工作流。**本节是给 Phase 2 时设计师 agent 自己用的 SOP**。

### 5.1 流程概览

```
┌────────────┐   ┌────────────┐   ┌────────────┐   ┌────────────┐
│ 1. 接收    │ → │ 2. 提取     │ → │ 3. 填实     │ → │ 4. 切图    │
│  设计稿    │   │   令牌      │   │   模板     │   │   与图标   │
└────────────┘   └────────────┘   └────────────┘   └────────────┘
                                                            │
                                                            ▼
                          ┌────────────┐   ┌────────────┐
                          │ 6. 交付    │ ← │ 5. 写组件   │
                          │   前端     │   │   规范     │
                          └────────────┘   └────────────┘
```

### 5.2 步骤 1 —— 接收设计稿

- 设计稿格式: Figma / Sketch / 蓝湖 / 静态图均可
- 必备: 5 个主要页面(首页/详情/发布/频道创建/通知)的高保真 + 关键弹层
- 必备: 设计令牌面板(色板/字号/间距/圆角/阴影单独一页)

**设计师 agent 动作**:
- 核对清单(本文件 §6.2 的"组件交付清单")是否齐全
- 不全 → 反馈用户补齐

### 5.3 步骤 2 —— 提取色板 → 写入 `app.wxss` 新令牌

1. 从设计稿的"颜色"面板提取所有色值
2. 与本文件 §2.7 的 [新增] token 清单对照:
   - 已规划 token → 把"待设计稿定"字段填实
   - 出现规划外的新色 → 补一条 [新增] token(命名遵循 `[角色]-[修饰]` 格式,如 `--warning-soft`)
3. 写入 `miniprogram/app.wxss` 的 `page { ... }` 块
4. **同时修正** §2.7 标记 [修正] 的 token(主要是 `--green` / `--success` / `--accent` / `--radius-lg`)
5. **同时废弃** §2.7 标记 [废弃] 的 token(在变量前加 `@deprecated` 注释,暂不删除以免破坏其他非社区页面)

### 5.4 步骤 3 —— 提取字号/间距/圆角/阴影 → 比对/更新令牌

- 字号: 按本文件 §1.2 的 6 个角色分配
- 间距: 按本文件 §1.6 的 8 个档位,新增 `--page-padding-x/y` / `--card-gap`
- 圆角: 按 §1.5 修正 `--radius-lg` 不再与 `--radius-md` 重复
- 阴影: 按 §1.7 新增 4 个 hero/card/cta 阴影 token

### 5.5 步骤 4 —— 切图与图标

**图标**:
- **优先 svg 矢量**(小程序对 svg 支持良好)
- iconfont 次之
- 单色 png 2x/3x 最后
- 必备清单见 §3.1 Icon

**图片**:
- 头像默认占位图(男女两版)
- 空状态插画(社区首页空/发布完成/网络错误)
- 加载占位骨架(无图片,代码生成)

**Tab 图标**:
- `channel-default.png` / `channel-active.png`,2x/3x
- 尺寸与活动 Tab 一致,视觉权重相同

### 5.6 步骤 5 —— 组件状态规范

对每个组件(Figma 组件变体 → 小程序 properties + states):
- 在 Figma 中创建组件,设置好所有变体
- 导出 props 对照表
- 给前端 agent 一份"组件 props 规范表"(见 §6.2 模板)

### 5.7 步骤 6 —— 给前端 agent 的对照表模板(Figma 图层名 → wxss 类名 → 组件 prop)

| Figma 图层名 | wxss 类名 | 组件 prop |
|--------------|-----------|-----------|
| `Hero / Title` | `.hero-title` | `<Title size="lg">` |
| `WaterfallCard` | `.waterfall-card` | `<WaterfallCard post={...} column={2}>` |
| `WaterfallCard / Image` | `.waterfall-image` | (内部 `ImageGrid`) |
| `ActionBar / Like` | `.action-like` | `<ActionBar.Liked liked={...}>` |
| ... | ... | ... |
