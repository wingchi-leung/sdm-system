# 真机访问部署操作文档（个人笔记本 + 域名）

本文档面向「用一台旧笔记本部署 backend，让微信小程序真机/体验版/正式版能访问」的场景。按顺序执行即可。

---

## 一、总览与耗时预估

| 阶段 | 内容 | 预估耗时 | 说明 |
|------|------|----------|------|
| **0** | 准备：域名、公网环境 | 0～2 天 | 无域名需购买/备案；无公网 IP 需申请或换网络 |
| **1** | 域名解析到笔记本（含 DDNS） | 15～30 分钟 | 有固定 IP 更快；用 DDNS 需装客户端 |
| **2** | 路由器端口转发 | 10～20 分钟 | 需能登录家里/公司路由器 |
| **3** | 笔记本上跑后端 + Nginx + HTTPS | 30～60 分钟 | 首次装 Nginx、申请证书会稍久 |
| **4** | 小程序 baseUrl + 微信合法域名 | 5～10 分钟 | 改代码、后台点几下 |
| **5** | 联调与验证 | 10～20 分钟 | 真机扫码测一遍流程 |

- **已有域名 + 有公网 IP（或可端口转发）**：整体约 **1～2 小时** 可跑通。  
- **需买域名 / 备案 / 申请公网 IP**：整体约 **1～3 天**（含等待）。

---

## 二、前置条件（必须先满足）

- [ ] **域名**：已购买且可解析（例如 `api.你的域名.com`）。  
  - 国内服务器若用国内域名，需备案；用个人笔记本+家宽一般用「解析到当前公网 IP」即可，具体以当地和运营商为准。  
- [ ] **公网可达**：笔记本所在网络有公网 IP，或能通过端口转发从外网访问到笔记本。  
  - 家宽：多数为动态 IP，需用 **DDNS**（见下）。  
  - 公司/学校网络：可能有防火墙，需管理员放行 80/443 或指定端口。  
- [ ] **笔记本**：可长期开机、运行 backend；建议 Windows 10/11 或 Windows Server。  
- [ ] **MySQL**：backend 所需的数据库已安装并建表（见 `backend/README.md`、`backend/table.sql`）。

---

## 三、阶段 1：域名解析到笔记本（约 15～30 分钟）

目标：让「你的域名」解析到你笔记本所在网络的公网 IP。

### 1.0 先判断：是家宽动态 IP 还是固定公网 IP？

**简单自测（约 5 分钟）：**

1. **看当前公网 IP**  
   用连接该宽带的电脑/手机，打开 [https://www.ip.cn](https://www.ip.cn) 或 [https://ip.sb](https://ip.sb)，记下显示的 IP（例如 `123.45.67.89`）。

2. **重启光猫/路由器**  
   拔电或管理页里点「重启」，等 2～5 分钟再连上网。

3. **再次打开上述网站**  
   再看一次当前公网 IP。

4. **对比两次 IP**  
   - **两次一样** → 可能是固定公网 IP，或只是短期内没变；可再隔一天或改天断网重连后再测一次。  
   - **两次不一样** → 就是**动态公网 IP**（家宽绝大多数是这种），需要用 DDNS（见下 1.2 方式 B）。

**按网络类型大致判断：**

| 网络类型           | 常见情况           | 建议做法           |
|--------------------|--------------------|--------------------|
| 家庭宽带（电信/联通/移动等） | 绝大多数是**动态公网 IP** | 用 **DDNS**        |
| 家庭宽带            | 少数地区/套餐有固定 IP   | 自测确认；不变就用 A 记录 |
| 公司/单位专线       | 常有固定 IP        | 问网管；有固定 IP 就填 A 记录 |
| 手机热点/4G/5G      | 一般为动态、且可能无入站 | 不适合做对外服务   |

**没有公网 IP 怎么办？**  
若你发现路由器 WAN 口 IP 和 ip.cn 显示的**不一致**（例如 WAN 是 `10.x.x.x`、`100.x.x.x`），说明在运营商大内网里，外网无法直接访问你家。可打运营商客服申请「公网 IP」（有的免费，有的要改套餐），或改用有公网 IP 的环境（如云服务器）做转发。

### 1.1 确认公网 IP

- 在笔记本浏览器打开 [https://www.ip.cn](https://www.ip.cn) 或 [https://ip.sb](https://ip.sb)，记下显示的 **公网 IP**。  
- 若为家宽，该 IP 可能会变，所以需要 DDNS（见 1.2）。

### 1.2 二选一：固定 IP 或 DDNS

**方式 A：有固定公网 IP**

- 登录你的 **域名服务商**（阿里云、腾讯云、Cloudflare 等）。  
- 添加 **A 记录**：  
  - 主机记录：`api`（则域名为 `api.你的域名.com`）或 `@`（根域名）。  
  - 记录值：你的公网 IP。  
  - TTL：600 或默认。  
- 保存后等待生效（约 5～10 分钟，最多几小时）。

**方式 B：动态 IP，用 DDNS（推荐家宽）**

1. 选一个 DDNS 服务（例如 **花生壳**、**No-IP**、**Dynu**、**DuckDNS** 等），注册并创建一条「域名」或「主机名」，得到类似 `xxx.ddns.net` 或你自有域名通过 CNAME 指向服务商。  
2. 在笔记本上安装该服务商的 **DDNS 客户端**，用账号登录，绑定「当前公网 IP → 你的主机名」。  
3. 若使用**自有域名**：在域名服务商处添加 **CNAME**，将 `api.你的域名.com` 指向 DDNS 提供的主机名（如 `xxx.ddns.net`）。  
4. 确认：在笔记本上 `ping api.你的域名.com`（或你的 DDNS 主机名），能解析到当前公网 IP 即可。

**耗时**：固定 IP 约 15 分钟；DDNS 安装+配置约 20～30 分钟。

## 四、阶段 2：路由器端口转发（约 10～20 分钟）

目标：外网访问「公网 IP:443」时，流量被转到笔记本的 443 端口（后续 Nginx 监听）。

1. 登录 **路由器管理页**（常见为 `192.168.1.1` 或 `192.168.0.1`）。  
2. 找到 **端口转发 / 虚拟服务器 / NAT** 等菜单。  
3. 新增规则：  
   - 外部端口：**443**（HTTPS）  
   - 内部 IP：**笔记本的局域网 IP**（如 `192.168.1.100`，在笔记本上 `ipconfig` 查看）  
   - 内部端口：**443**  
   - 协议：TCP  
4. 可选：再添加一条 **80** → 笔记本 IP:80，便于用 HTTP 做证书校验（Let's Encrypt）。  
5. 保存。若笔记本 IP 会变，建议在路由器里给该笔记本做 **DHCP 静态绑定**（按 MAC 固定 IP）。

**耗时**：约 10～20 分钟。

## 五、阶段 3：笔记本上后端 + Nginx + HTTPS（约 30～60 分钟）

### 5.1 确认后端可本机访问（约 5 分钟）

在笔记本上：

```bash
cd D:\sdm-system 2\sdm-system\backend
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

浏览器访问 `http://localhost:8000/docs`，能打开即正常。生产环境可去掉 `--reload`，或用进程守护（如 NSSM）保活，此处先保证能跑通。

### 5.2 安装 Nginx（约 10 分钟）

- 从 [Nginx for Windows](https://nginx.org/en/download.html) 下载稳定版，解压到如 `C:\nginx`。  
- 确认 `C:\nginx\nginx.exe` 能启动（命令行运行一次即可）。  
- 若 80/443 被占用，先改 `conf\nginx.conf` 里的 `listen` 端口或关闭占用端口的程序。

### 5.3 申请 HTTPS 证书（约 10～20 分钟）

小程序要求 **HTTPS**，推荐用 **Let's Encrypt** 免费证书。

- **Windows 上**可用：  
  - [win-acme](https://www.win-acme.com/)（带图形界面），或  
  - 在**同一局域网内**的一台 Linux 或 WSL 上用 **certbot** 申请，把得到的 `fullchain.pem`、`privkey.pem` 拷到笔记本（如 `C:\nginx\ssl\`）。  
- 申请前确保：  
  - 域名已解析到当前公网 IP；  
  - 路由器 80 已转发到笔记本 80；  
  - Nginx 先临时监听 80，用于 certbot 的 HTTP 校验（或用 certbot 的 standalone 模式时临时关掉 Nginx）。  
- 证书得到后放在例如：  
  - `C:\nginx\ssl\你的域名\fullchain.pem`  
  - `C:\nginx\ssl\你的域名\privkey.pem`

**耗时**：首次约 10～20 分钟（含下载工具、解析生效、校验通过）。

### 5.4 配置 Nginx 反向代理（约 10 分钟）

在 `C:\nginx\conf\` 下新建或修改站点配置（例如 `conf\vhost\api.conf`），内容示例（把 `api.你的域名.com` 和证书路径改成你的）：

```nginx
server {
    listen 443 ssl;
    server_name api.你的域名.com;

    ssl_certificate     C:/nginx/ssl/你的域名/fullchain.pem;
    ssl_certificate_key C:/nginx/ssl/你的域名/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

在 `nginx.conf` 的 `http { ... }` 里增加一行：`include vhost/*.conf;`（或你实际放的路径）。  
重载 Nginx：`nginx -s reload`。

此时应能通过 `https://api.你的域名.com/docs` 访问到后端 Swagger（证书告警可先忽略，若证书正确会消失）。

**耗时**：约 10 分钟。

### 5.5 后端生产环境建议（可选，约 5 分钟）

- 在 `backend\.env` 中设置强随机 `JWT_SECRET`。  
- 确认 MySQL 连接、`table.sql` 已执行、管理员账号已建好（见 `backend/README.md`）。  
- 若希望 backend 开机自启：可用 NSSM 把 `uv run uvicorn app.main:app --host 0.0.0.0 --port 8000` 注册为 Windows 服务。

## 六、阶段 4：小程序 baseUrl + 微信合法域名（约 5～10 分钟）

### 6.1 改小程序 API 基地址

在项目里修改 `miniprogram/utils/api.js`，将：

```js
const baseUrl = 'http://localhost:8000/api/v1';
```

改为（域名和路径与 Nginx 一致）：

```js
const baseUrl = 'https://api.你的域名.com/api/v1';
```

若你 Nginx 的 `location` 是 `location /api/ { ... }` 且 `proxy_pass http://127.0.0.1:8000;`，则后端实际收到的是 `/api/v1/...`，与当前 backend 路由一致，无需改后端。若你用了 `location /`，则 baseUrl 保持 `https://api.你的域名.com/api/v1` 即可。

### 6.2 配置微信 request 合法域名

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → 你的小程序。  
2. **开发** → **开发管理** → **开发设置** → **服务器域名**。  
3. 在 **request 合法域名** 中点击「修改」，添加：  
   `https://api.你的域名.com`  
   （只填域名，不要路径、不要末尾斜杠）。  
4. 保存（会校验域名 HTTPS 可访问，需 Nginx 已配好且证书有效）。

**耗时**：约 5～10 分钟。

---

## 七、阶段 5：联调与验证（约 10～20 分钟）

1. **本机**：再次确认 `https://api.你的域名.com/docs` 可打开。  
2. **真机**：  
   - 微信开发者工具中上传「体验版」，或直接真机预览。  
   - 手机连网（可与笔记本不同网络，能访问公网即可）。  
   - 打开小程序：活动列表、登录、报名等走一遍，确认无「不在合法域名列表中」等报错。  
3. 若有问题：  
   - 看微信开发者工具控制台或手机 vConsole 的网络请求是否指向 `https://api.你的域名.com`；  
   - 看 Nginx、uvicorn 的访问/错误日志。

**耗时**：约 10～20 分钟。

## 八、检查清单（真机可用的最小集合）

- [ ] 域名已解析到当前公网 IP（或通过 DDNS 动态更新）。  
- [ ] 路由器 443（及可选 80）已转发到笔记本对应端口。  
- [ ] 笔记本上 backend 监听 `0.0.0.0:8000` 且可访问。  
- [ ] Nginx 监听 443，证书有效，反代到 `http://127.0.0.1:8000`。  
- [ ] `miniprogram/utils/api.js` 的 baseUrl 为 `https://api.你的域名.com/api/v1`。  
- [ ] 微信公众平台 request 合法域名已添加 `https://api.你的域名.com`。  
- [ ] 真机实测：列表、登录、报名等接口均正常。

## 九、常见问题

- **真机提示「不在 request 合法域名列表」**：检查公众平台填的域名是否与 baseUrl 的域名完全一致（含 HTTPS）、是否保存成功；有时需等几分钟生效。  
- **请求超时**：检查路由器端口转发、笔记本防火墙是否放行 443、backend 是否在跑。  
- **证书不受信任**：确保用的是 Let's Encrypt 等正规证书，且域名与证书 CN/SAN 一致。  
- **家宽封 80/443**：部分运营商封锁家用 80/443，需改用其他端口（如 8443）并在 Nginx 监听 8443、路由器转发 8443；微信合法域名仍填 `https://api.你的域名.com`，不写端口（需该域名 443 或 8443 在 Nginx 上实际提供 HTTPS）。
