# 备份与恢复演练手册

> 更新时间：2026-05-09
> 适用范围：旧电脑正式部署（Docker + MySQL + Cloudflare Tunnel）

## 一、目标

这份手册解决两个问题：

1. 旧电脑坏了，多久能恢复
2. 现在的备份到底能不能真的用

目标不是“把文件复制一下”，而是做到：

- 能导出一份完整备份
- 能在另一个目录或另一台机器恢复
- 恢复后服务能重新对外提供访问

---

## 二、必须备份的内容

### 1. 数据库

来源：

- Docker MySQL 容器中的业务数据

建议形式：

- `mysqldump` SQL 文件

### 2. 上传文件

来源：

- `backend/uploads/`

说明：

- 活动海报、头像等用户上传文件都在这里

### 3. 支付证书

来源：

- `secrets/wechat_pay/`

说明：

- 没有它，微信支付基本就恢复不起来

### 4. 正式配置

来源：

- 根目录 `.env`

说明：

- 没有它，容器能起来，业务未必能跑

### 5. Tunnel 配置

来源：

- `tunnel/config.yml`
- `tunnel/certs/`

说明：

- 没有它，公网入口可能恢复不了

---

## 三、不需要放进备份的内容

- `frontend/node_modules/`
- `frontend/build/`
- `backend/.venv/`
- `htmlcov/`
- `.pytest_cache/`
- Git 历史对象

这些都可以重新生成，不值得占备份空间。

---

## 四、建议的备份目录结构

每次备份生成一个目录，例如：

```text
D:\sdm-backups\20260509-213000\
├── db\
│   └── sdm_db.sql
├── files\
│   └── uploads\
├── secrets\
│   └── wechat_pay\
├── config\
│   ├── .env
│   ├── docker-compose.yml
│   └── tunnel\
└── manifest.txt
```

---

## 五、自动备份脚本

仓库已补充脚本：

- [scripts/backup-production.ps1](</D:/sdm-system 2/sdm-system/scripts/backup-production.ps1>)

默认会做这些事：

1. 检查 `.env`
2. 检查 Docker MySQL 容器
3. 导出数据库 SQL
4. 复制 `backend/uploads/`
5. 复制 `secrets/wechat_pay/`
6. 复制 `.env`、`docker-compose.yml`、`tunnel/`
7. 输出 `manifest.txt`

执行方式：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\backup-production.ps1
```

可自定义输出目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\backup-production.ps1 -BackupRoot "D:\sdm-backups"
```

## 六、恢复演练步骤

### 步骤 1：准备恢复目录

例如：

```powershell
New-Item -ItemType Directory -Path D:\sdm-restore-test
```

### 步骤 2：复制恢复所需文件

从备份中恢复：

- `config/.env` -> 项目根目录 `.env`
- `config/docker-compose.yml` -> 项目根目录
- `config/tunnel/` -> 项目根目录 `tunnel/`
- `secrets/wechat_pay/` -> 项目根目录 `secrets/wechat_pay/`
- `files/uploads/` -> 项目根目录 `backend/uploads/`

### 步骤 3：启动基础容器

```powershell
docker compose up -d mysql
```

等待 MySQL 就绪。

### 步骤 4：恢复数据库

```powershell
Get-Content D:\你的备份目录\db\sdm_db.sql | docker exec -i sdm-mysql mysql -u root -p你的root密码 sdm_db
```

如果是新环境，先确认数据库已创建。

### 步骤 5：启动全量服务

```powershell
docker compose up -d
```

### 步骤 6：验证恢复结果

至少检查：

- API 文档页可打开
- Web 后台可登录
- 小程序能正常请求活动列表
- 海报/头像能打开
- 支付订单查询接口正常

---

## 七、恢复验收清单

- [ ] `docker compose ps` 全部核心容器正常
- [ ] `https://api.你的域名.com/docs` 可访问
- [ ] `https://web.你的域名.com` 可访问
- [ ] 活动列表正常
- [ ] 管理员登录正常
- [ ] 已上传图片能正常打开
- [ ] 小程序登录正常
- [ ] 报名流程正常
- [ ] 支付订单查询正常

---

## 八、建议频率

- 每天自动备份一次
- 每周人工看一次备份目录
- 每月做一次恢复演练
- 改域名、改支付证书、改 MySQL 密码后立刻重做备份

---

## 九、当前风险提醒

现阶段最容易让恢复失败的点有 4 个：

1. 只备份了数据库，没有备份上传文件
2. 只备份了 `.env`，没有备份支付证书
3. Tunnel 证书或配置没备份，导致公网入口起不来
4. 从没做过恢复演练，真正出事时才第一次恢复

所以你现在最重要的动作，不是继续“想怎么备份更好”，而是先按这份文档完整跑通一次。
