# SDM 系统 — 鉴权技术文档

本文档完整说明系统的鉴权设计、密码传输安全、以及各端实现要点。

---

## 一、鉴权架构概览

### 1.1 谁需要鉴权

| 角色 | 说明 | 鉴权方式 |
|------|------|----------|
| **管理员** | 发布活动、管理后台、现场签到等 | 用户名 + 密码登录，服务端签发 JWT，后续请求带 `Authorization: Bearer <token>` |
| **普通用户（C 端）** | 浏览活动、报名 | 无需登录，报名/签到接口不校验 JWT |
| **报名/签到** | 仅校验「是否已报名」「是否已签到」 | 不依赖管理员身份，仅校验业务数据 |

### 1.2 两张用户表（切勿混淆）

| 表名 | 用途 | 是否参与登录 |
|------|------|--------------|
| **admin_user** | 管理员账号：username + password_hash | **是**，登录鉴权只用此表 |
| **user** | 参与人档案：姓名、证件号、手机、拉黑等 | **否**，不存密码，不用于登录 |

登录接口只查 **admin_user**，校验 `password_hash`；**user** 表仅用于业务数据与黑名单。

### 1.3 鉴权流程（管理员）

```
1. 客户端 POST /api/v1/auth/login
   Body: { "username": "admin", "password": "xxx" }

2. 服务端：
   - 在 admin_user 中按 username 查记录
   - 用 bcrypt 校验 password 与 password_hash
   - 通过则生成 JWT（sub=管理员 id, role=admin, exp=过期时间）
   - 返回 { "access_token": "<JWT>", "token_type": "bearer" }

3. 客户端保存 token（Web：localStorage；Flutter：SharedPreferences）

4. 后续需鉴权的请求：
   Header: Authorization: Bearer <access_token>
   服务端 get_current_admin 依赖项解析 JWT，校验 role=admin、exp 未过期，得到 admin_id
```

需要 **Bearer Token** 的接口目前仅有：**创建活动** `POST /api/v1/activities/`。

## 二、密码传输与「F12 看到明文」问题

### 2.1 现象

在浏览器开发者工具（F12）→ Network → 选中登录请求 → Payload 中可以看到：

```json
{ "username": "admin", "password": "你的密码" }
```

即密码以明文形式出现在请求体里。

### 2.2 为何会这样

- 使用「用户名 + 密码」登录时，服务端必须能**验证密码**，因此客户端**必须**把密码（或服务端能验证的等价信息）发到服务端。
- 请求体在 F12 里是**解密后**展示的，所以只要密码在 Body 里，开发者工具里就会看到明文。这是密码登录的固有特点，不是实现错误。

### 2.3 真正要防的是什么

要防范的是**传输过程被窃听**（例如同一 WiFi 下抓包），而不是「开发者在自己的 F12 里看到自己的请求」。

| 场景 | 风险 | 正确做法 |
|------|------|----------|
| **生产环境用 HTTP** | 密码在网络上明文传输，可被窃听、篡改 | **禁止**：生产环境登录必须走 **HTTPS** |
| **生产环境用 HTTPS** | 请求体在传输层加密，抓包看不到明文密码 | 可接受；F12 里仍会看到明文是正常现象 |
| **本地开发 localhost** | 一般仅本机访问，风险可控 | 可用 HTTP；若需更谨慎可本地配 HTTPS |

结论：**生产环境必须使用 HTTPS**；仅在开发环境（如 localhost）允许 HTTP。

### 2.4 本系统已做的安全增强

1. **生产环境禁止 HTTP 登录**  
   后端提供配置项 `REQUIRE_HTTPS_FOR_LOGIN`。为 `true` 时，若检测到请求未经 HTTPS（如通过 `X-Forwarded-Proto`），则直接拒绝登录，避免密码在公网明文传输。

2. **登录限流**  
   对 `POST /api/v1/auth/login` 做按 IP 的限流（如每分钟最多 N 次），降低暴力破解风险。

3. **前端/App 安全提示**  
   - 当配置的 API 地址为 **http 且非 localhost** 时，登录页会提示：「当前连接未加密，密码可能被窃听，请仅在可信网络使用或改用 HTTPS。」
   - 提醒部署时使用 HTTPS。

4. **不在日志或响应中输出密码**  
   服务端从不把密码写入日志或 API 响应；仅做 bcrypt 校验后丢弃。

## 三、JWT 与 Token 使用

### 3.1 JWT 内容

- `sub`：管理员 id（admin_user.id）
- `role`：固定为 `"admin"`
- `exp`：过期时间（默认 7 天，由 `JWT_EXPIRE_MINUTES` 配置）

密钥由 `JWT_SECRET` 配置；**生产环境必须修改**，且不要提交到代码库。

### 3.2 校验方式（后端）

- 依赖项：`get_current_admin`（见 `app/api/deps.py`）
- 从请求头取 `Authorization: Bearer <token>`
- 用 `JWT_SECRET` 解码并校验签名、`role == "admin"`、`exp` 未过期
- 通过则返回 `admin_id`（int），供创建活动等接口使用

### 3.3 各端如何带 Token

| 端 | 存储 | 发送方式 |
|----|------|----------|
| **管理端 Web（React）** | localStorage `admin_token` | 请求头 `Authorization: Bearer ${token}`，由统一 `apiRequest` 自动附带 |
| **C 端 App（Flutter）** | SharedPreferences `admin_access_token` | 发布活动等接口在请求头带 `Authorization: Bearer $accessToken` |

## 四、接口鉴权一览

| 接口 | 是否需要管理员 Token | 说明 |
|------|----------------------|------|
| `POST /api/v1/auth/login` | 否 | 管理员登录入口 |
| `POST /api/v1/auth/user-login` | 否 | 普通用户登录（手机+密码） |
| `POST /api/v1/auth/wechat-login` | 否 | 微信小程序授权登录（Body: `{ "code": "..." }`，需配置 WECHAT_APPID/SECRET） |
| `GET /api/v1/activities/` | 否 | 活动列表（含 C 端） |
| `GET /api/v1/activities/unstarted/` | 否 | 未开始活动 |
| `POST /api/v1/activities/` | **是** | 创建活动 |
| `PUT /api/v1/activities/{id}/status` | 否 | 更新状态（按需可改为需鉴权） |
| `GET /api/v1/participants/{id}/` | 否 | 某活动参与人列表 |
| `POST /api/v1/participants/` | 否 | 报名 |
| `POST /api/v1/checkins/` | 否 | 签到 |
| `GET /api/v1/checkins/` | 否 | 签到记录列表（统计用，按需可改为需鉴权） |
| `GET /api/v1/activities/{id}/checkins/` | 否 | 某活动签到列表 |
| `GET /api/v1/users/` 等 | 否 | 用户列表等（按需可改为需鉴权） |

当前仅 **创建活动** 强制要求管理员 Token；其余接口可根据需求再收紧。

## 五、配置项汇总（鉴权与安全）

| 配置项 | 说明 | 默认值 | 生产建议 |
|--------|------|--------|----------|
| `JWT_SECRET` | JWT 签名密钥 | `change-me-in-production` | **必须**改为随机长字符串，且不提交仓库 |
| `JWT_EXPIRE_MINUTES` | Token 有效分钟数 | 10080（7 天） | 可按策略缩短 |
| `REQUIRE_HTTPS_FOR_LOGIN` | 是否仅允许 HTTPS 登录 | `false` | 生产设为 `true`；反向代理需正确设置 `X-Forwarded-Proto` |
| `WECHAT_APPID` | 微信小程序 AppID（微信授权登录） | 未设置 | 启用小程序登录时在 .env 中配置 |
| `WECHAT_SECRET` | 微信小程序 AppSecret | 未设置 | 启用小程序登录时在 .env 中配置；勿提交仓库 |

## 六、部署与运维提醒

1. **生产环境**  
   - 全站使用 **HTTPS**。  
   - 设置 `REQUIRE_HTTPS_FOR_LOGIN=true`。  
   - 使用强 `JWT_SECRET`，并定期评估更换策略。

2. **反向代理**  
   若前端/API 经 Nginx 等反向代理，需正确传递 `X-Forwarded-Proto: https`，以便后端判断为 HTTPS。

3. **首次管理员**  
   见 `backend/docs/鉴权说明.md`：建表、用脚本生成 password_hash、插入 admin_user。

## 七、本次实现改动摘要

针对「密码明文传输」与鉴权闭环，已做如下实现：

| 项 | 说明 |
|----|------|
| **鉴权技术文档** | 新增 `docs/鉴权技术文档.md`，说明鉴权架构、admin_user vs user、密码传输与 F12、HTTPS 要求、JWT、接口鉴权一览、配置项、部署提醒。 |
| **后端：HTTPS 约束** | 配置项 `REQUIRE_HTTPS_FOR_LOGIN`（默认 `false`）。为 `true` 时，通过 `X-Forwarded-Proto` 或请求 scheme 判断，非 HTTPS 则拒绝登录（403）。 |
| **后端：登录限流** | 按 IP 限流：`LOGIN_RATE_LIMIT_COUNT`（默认 10）次 / `LOGIN_RATE_LIMIT_WINDOW_SECONDS`（默认 60）秒，超出返回 429。 |
| **前端：登录与 Token** | 新增 `/login` 登录页；Token 存 localStorage；`apiRequest` 统一附带 `Authorization: Bearer <token>`；创建活动等需鉴权路由用 `ProtectedRoute` 保护，未登录跳转 `/login?redirect=...`。 |
| **前端：HTTP 安全提示** | 当 API 为 http 且非 localhost/127.0.0.1 时，登录页展示安全提示：密码可能被窃听，生产环境请用 HTTPS。 |
| **Flutter：HTTP 安全提示** | 当 `baseUrl` 为 http 且非 localhost/127.0.0.1/10.0.2.2 时，登录页展示相同含义的安全提示。 |
| **统计页接口** | 后端新增 `GET /api/v1/checkins/`（支持 skip、limit、activity_id），返回签到记录及活动名称；前端统计页改为调用该接口，不再请求不存在的 `/records`。 |

生产环境建议：启用 HTTPS，设置 `REQUIRE_HTTPS_FOR_LOGIN=true`，并修改 `JWT_SECRET`。
