# 微信支付开发方案

## 一、概述

本文档描述 SDM 系统中微信支付功能的实现方案，用于活动报名时的费用支付。

### 1.1 业务需求

- 活动创建者可设置活动的建议费用
- 用户报名时可支付等于或高于建议费用的金额
- 支付成功后自动创建报名记录
- 暂不支持退款功能

### 1.2 技术选型

- 支付方式：微信支付 JSAPI（小程序支付）
- SDK：wechatpayv3 (Python)
- 数据库：MySQL

## 二、配置说明

### 2.1 环境变量配置

在 `backend/.env` 文件中添加以下配置：

```env
# 微信小程序配置（已有）
WECHAT_APPID=你的小程序AppID
WECHAT_SECRET=你的小程序Secret

# 微信支付配置
WECHAT_PAY_MCH_ID=商户号
WECHAT_PAY_API_V3_KEY=APIv3密钥（32位）
WECHAT_PAY_SERIAL_NO=商户API证书序列号
WECHAT_PAY_PRIVATE_KEY_PATH=私钥文件绝对路径
WECHAT_PAY_NOTIFY_URL=支付回调地址
```

### 2.2 配置项说明

| 配置项 | 说明 | 获取方式 |
|--------|------|----------|
| `WECHAT_PAY_MCH_ID` | 商户号 | 微信支付商户平台 → 账户中心 |
| `WECHAT_PAY_API_V3_KEY` | APIv3密钥 | 商户平台 → 账户中心 → API安全 → 设置APIv3密钥 |
| `WECHAT_PAY_SERIAL_NO` | 证书序列号 | 商户平台 → 账户中心 → API安全 → 查看证书 |
| `WECHAT_PAY_PRIVATE_KEY_PATH` | 私钥路径 | 申请证书时下载的 `apiclient_key.pem` 文件路径 |
| `WECHAT_PAY_NOTIFY_URL` | 回调地址 | 需为 HTTPS 地址，格式：`https://域名/api/v1/payments/notify` |

### 2.3 证书文件存放

```bash
# 建议目录结构（不要放在项目代码目录内）
/etc/wechat_pay/
├── apiclient_key.pem      # 商户私钥
└── apiclient_cert.pem     # 商户证书（可选）
```

### 2.4 安全注意事项

- **切勿提交** `.env` 文件和 `.pem` 私钥文件到代码仓库
- 私钥文件权限应设为 `600`（仅所有者可读写）
- 生产环境使用密钥管理服务（如 KMS）
- 定期轮换 APIv3 密钥

## 三、数据库设计
 
### 3.1 数据库迁移 SQL

```sql
-- 1. 修改 activity 表
ALTER TABLE activity
ADD COLUMN suggested_fee INT DEFAULT 0 COMMENT '建议费用（分）',
ADD COLUMN require_payment INT DEFAULT 0 COMMENT '是否需要支付：0-否 1-是';

-- 2. 修改 activity_participants 表
ALTER TABLE activity_participants
ADD COLUMN payment_status INT DEFAULT 0 COMMENT '0-无需支付 1-待支付 2-已支付',
ADD COLUMN payment_order_id INT DEFAULT NULL COMMENT '支付订单ID',
ADD COLUMN paid_amount INT DEFAULT 0 COMMENT '实际支付金额（分）';

-- 3. 创建 payment_order 表
CREATE TABLE payment_order (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tenant_id INT NOT NULL DEFAULT 1,
    order_no VARCHAR(64) NOT NULL UNIQUE COMMENT '商户订单号',
    transaction_id VARCHAR(64) DEFAULT NULL COMMENT '微信交易号',
    activity_id INT NOT NULL COMMENT '活动ID',
    user_id INT DEFAULT NULL COMMENT '用户ID',
    participant_id INT DEFAULT NULL COMMENT '参与者ID',
    suggested_fee INT NOT NULL COMMENT '建议费用（分）',
    actual_fee INT NOT NULL COMMENT '实际支付金额（分）',
    status INT DEFAULT 0 COMMENT '0-待支付 1-成功 2-失败 3-关闭',
    openid VARCHAR(64) DEFAULT NULL COMMENT '付款用户openid',
    prepay_id VARCHAR(128) DEFAULT NULL COMMENT '预支付ID',
    paid_at DATETIME DEFAULT NULL COMMENT '支付成功时间',
    expire_at DATETIME NOT NULL COMMENT '过期时间',
    callback_raw VARCHAR(2000) DEFAULT NULL COMMENT '回调原始数据',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_tenant_id (tenant_id),
    INDEX idx_order_no (order_no),
    INDEX idx_transaction_id (transaction_id),
    INDEX idx_activity_id (activity_id),
    INDEX idx_user_id (user_id),
    INDEX idx_participant_id (participant_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='支付订单表';
```

## 四、API 接口

### 4.1 创建支付订单

**请求**
```
POST /api/v1/payments/create
Authorization: Bearer <token>
Content-Type: application/json
```

**参数**
```json
{
  "activity_id": 1,
  "participant_name": "张三",
  "phone": "13800138000",
  "identity_number": "身份证号（可选）",
  "actual_fee": 10000
}
```

**响应**
```json
{
  "order_no": "SDM20240321123456ABCD1234",
  "activity_id": 1,
  "suggested_fee": 10000,
  "actual_fee": 10000,
  "status": 0,
  "payment_params": {
    "appId": "wx1234567890",
    "timeStamp": "1711000000",
    "nonceStr": "abc123",
    "package": "prepay_id=wx123",
    "signType": "RSA",
    "paySign": "签名值"
  }
}
```

### 4.2 支付回调

**请求**
```
POST /api/v1/payments/notify
Content-Type: application/json
```

微信支付服务器会发送加密的回调数据，系统自动解密并处理。

**响应**
```json
{
  "code": "SUCCESS",
  "message": "成功"
}
```

### 4.3 查询订单

**请求**
```
GET /api/v1/payments/order/{order_no}
Authorization: Bearer <token>
```

**响应**
```json
{
  "id": 1,
  "order_no": "SDM20240321123456ABCD1234",
  "transaction_id": "微信交易号",
  "activity_id": 1,
  "user_id": 10,
  "participant_id": 100,
  "suggested_fee": 10000,
  "actual_fee": 10000,

## 五、支付流程

```

       │                   │                   │                   │

### 5.2 小程序端代码示例

```javascript
// 1. 创建支付订单
const orderData = await api.createPaymentOrder({
  activity_id: activityId,
  participant_name: name,
  phone: phone,
  actual_fee: actualFee,  // 单位：分
});

// 2. 调起微信支付
wx.requestPayment({
  timeStamp: orderData.payment_params.timeStamp,
  nonceStr: orderData.payment_params.nonceStr,
  package: orderData.payment_params.package,
  signType: orderData.payment_params.signType,
  paySign: orderData.payment_params.paySign,
  success: () => {
    wx.showToast({ title: '支付成功', icon: 'success' });
  },
  fail: (err) => {
    if (err.errMsg.includes('cancel')) {
      wx.showToast({ title: '已取消支付', icon: 'none' });
    } else {
      wx.showToast({ title: '支付失败', icon: 'none' });
    }
  },
});
```

## 六、错误处理

### 6.1 常见错误码

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 400 | 参数错误 | 检查请求参数 |
| 401 | 未登录 | 引导用户登录 |
| 403 | 无权限 | 检查用户权限 |
| 404 | 活动不存在 | 检查活动ID |
| 500 | 服务器错误 | 查看后端日志 |

### 6.2 支付异常处理

| 场景 | 处理方式 |
|------|----------|
| 用户取消支付 | 订单保持待支付状态，可重新发起 |
| 支付超时 | 订单自动关闭（默认30分钟） |
| 网络异常 | 用户可通过订单查询确认支付状态 |
| 重复支付 | 通过订单号唯一性约束避免 |
