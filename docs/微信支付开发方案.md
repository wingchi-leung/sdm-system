# 微信支付实现说明

本文档记录 SDM 系统中微信支付功能的实现情况、技术架构及开发环境配置。

---

## 一、概述

### 1.1 功能说明

微信支付功能用于活动报名时的费用支付，用户通过小程序完成支付后自动创建报名记录。

### 1.2 业务规则

- 活动创建者可设置活动的建议费用
- 用户报名时可支付等于或高于建议费用的金额
- 支付成功后自动创建报名记录
- 暂不支持退款功能

## 四、代码实现

### 4.1 后端文件结构

| 文件路径 | 功能说明 |
|----------|----------|
| `backend/app/services/wechat_pay.py` | 微信支付服务封装，单例模式 |
| `backend/app/api/v1/endpoints/payments.py` | 支付 API 端点 |
| `backend/app/models/payment.py` | Pydantic 请求/响应模型 |
| `backend/app/crud/crud_payment.py` | 支付订单数据库 CRUD |
| `backend/app/core/config.py` | 支付配置项定义 |
| `backend/app/schemas.py` | PaymentOrder ORM 模型 |

### 4.2 WeChatPayService 核心方法

| 方法 | 功能 |
|------|------|
| `create_jsapi_order()` | 创建 JSAPI 支付订单（统一下单） |
| `get_mini_program_payment_params()` | 获取小程序调起支付的签名参数 |
| `query_order()` | 查询订单状态 |
| `close_order()` | 关闭订单 |
| `decrypt_callback()` | 解密支付回调通知 |

### 4.3 小程序支付流程

支付调用位于 `miniprogram/pages/register/register.js`：

1. 调用 `api.createPaymentOrder()` 创建订单
2. 获取 `payment_params` 支付参数
3. 调用 `wx.requestPayment()` 调起微信支付
4. 支付成功后跳转结果页面

## 五、数据库设计

### 5.1 payment_order 表（已实现）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键 |
| `tenant_id` | INT | 租户ID |
| `order_no` | VARCHAR(64) | 商户订单号（唯一） |
| `transaction_id` | VARCHAR(64) | 微信交易号 |
| `activity_id` | INT | 活动ID |
| `user_id` | INT | 用户ID |
| `participant_id` | INT | 参与者ID |
| `suggested_fee` | INT | 建议费用（分） |
| `actual_fee` | INT | 实际支付金额（分） |
| `status` | INT | 状态：0-待支付 1-成功 2-失败 3-关闭 |
| `openid` | VARCHAR(64) | 付款用户 openid |
| `prepay_id` | VARCHAR(128) | 预支付ID |
| `paid_at` | DATETIME | 支付成功时间 |
| `expire_at` | DATETIME | 过期时间 |
| `callback_raw` | VARCHAR(2000) | 回调原始数据 |

### 5.2 订单状态流转

```
待支付(0) ──→ 成功(1)   [支付回调成功]
         ──→ 失败(2)   [支付失败]
         ──→ 关闭(3)   [超时关闭/手动关闭]
```

## 八、开发环境配置

### 8.1 Cloudflare Tunnel 配置

开发环境使用 Cloudflare Tunnel 实现内网穿透，使微信支付服务器能够访问本地后端。

**配置信息：**

| 配置项 | 值 |
|--------|-----|
| Tunnel 名称 | `sdm-dev` |
| Tunnel ID | `ec4f500e-7a1c-4e64-a506-59540c52e352` |
| 公网地址 | `https://api.ilovegenerator.online` |
| 本地映射 | `localhost:8000` |
| 配置文件 | `C:\Users\g0132\.cloudflared\config.yml` |

**配置文件内容：**

```yaml
tunnel: ec4f500e-7a1c-4e64-a506-59540c52e352
credentials-file: C:\Users\g0132\.cloudflared\ec4f500e-7a1c-4e64-a506-59540c52e352.json

ingress:
  - hostname: api.ilovegenerator.online
    service: http://localhost:8000
  - service: http_status:404
```

**常用命令：**

```powershell
# 启动 Tunnel
cloudflared tunnel run sdm-dev

# 查看 Tunnel 列表
cloudflared tunnel list

# 查看 Tunnel 信息
cloudflared tunnel info sdm-dev
```

**注意：** 每次重启电脑后需要重新运行 `cloudflared tunnel run sdm-dev` 启动隧道。

### 8.2 微信支付环境变量

在 `backend/.env` 中配置：

```env
# 微信支付配置
WECHAT_PAY_MCH_ID=商户号
WECHAT_PAY_API_V3_KEY=APIv3密钥（32位）
WECHAT_PAY_SERIAL_NO=商户API证书序列号
WECHAT_PAY_PRIVATE_KEY_PATH=私钥文件绝对路径
WECHAT_PAY_NOTIFY_URL=https://api.ilovegenerator.online/api/v1/payments/notify
```

**配置项获取方式：**

| 配置项 | 获取位置 |
|--------|----------|
| 商户号 | 微信支付商户平台 → 账户中心 |
| APIv3密钥 | 商户平台 → 账户中心 → API安全 → 设置APIv3密钥 |
| 证书序列号 | 商户平台 → 账户中心 → API安全 → 查看证书 |
| 私钥文件 | 申请证书时下载的 `apiclient_key.pem` |

### 8.3 安全注意事项

- **切勿提交** `.env` 文件和 `.pem` 私钥文件到代码仓库
- 私钥文件权限应设为仅所有者可读写
- 生产环境使用密钥管理服务（如 KMS）
- 定期轮换 APIv3 密钥
