# 权限系统重构方案（RBAC）

## 1. 数据模型

### 核心表结构

```sql
-- 权限表（系统预定义）
CREATE TABLE permission (
  id INT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(64) UNIQUE NOT NULL,  -- 'activity.create', 'activity.edit', 'activity.delete'
  name VARCHAR(100) NOT NULL,         -- '创建活动', '编辑活动', '删除活动'
  resource VARCHAR(32) NOT NULL,      -- 'activity', 'user', 'report'
  action VARCHAR(32) NOT NULL,        -- 'create', 'edit', 'delete', 'view'
  description TEXT
);

-- 角色表
CREATE TABLE role (
  id INT PRIMARY KEY AUTO_INCREMENT,
  tenant_id INT NOT NULL,
  name VARCHAR(64) NOT NULL,          -- '活动管理员', '签到员', '财务'
  is_system TINYINT DEFAULT 0,        -- 1=系统预设角色，0=自定义角色
  description TEXT,
  UNIQUE KEY (tenant_id, name)
);

-- 角色-权限关联表
CREATE TABLE role_permission (
  role_id INT NOT NULL,
  permission_id INT NOT NULL,
  PRIMARY KEY (role_id, permission_id)
);

-- 用户-角色关联表
CREATE TABLE user_role (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  role_id INT NOT NULL,
  tenant_id INT NOT NULL,
  scope_type VARCHAR(32),             -- NULL=全局, 'activity_type', 'activity'
  scope_id INT,                       -- 活动类型ID 或 活动ID
  UNIQUE KEY (user_id, role_id, scope_type, scope_id)
);
```

## 3. 预设角色

```sql
-- 超级管理员（全部权限）
INSERT INTO role (name, is_system) VALUES ('超级管理员', 1);

-- 活动管理员（管理活动和报名）
INSERT INTO role (name, is_system) VALUES ('活动管理员', 1);
INSERT INTO role_permission (role_id, permission_id)
SELECT 2, id FROM permission WHERE code IN (
  'activity.create', 'activity.edit', 'activity.delete', 'activity.view',
  'participant.view', 'participant.export', 'checkin.manage'
);

-- 签到员（仅签到）
INSERT INTO role (name, is_system) VALUES ('签到员', 1);
INSERT INTO role_permission (role_id, permission_id)
SELECT 3, id FROM permission WHERE code IN ('checkin.manage', 'participant.view');

-- 财务（查看支付数据）
INSERT INTO role (name, is_system) VALUES ('财务', 1);
INSERT INTO role_permission (role_id, permission_id)
SELECT 4, id FROM permission WHERE code IN ('participant.view', 'participant.export');
```

## 6. 优势

✅ **灵活扩展**：新增权限只需插入数据，不改代码
✅ **细粒度控制**：可以精确到单个活动
✅ **商业成熟**：RBAC 是行业标准
✅ **支持自定义角色**：租户可以创建自己的角色
✅ **向后兼容**：可以保留"活动类型授权"作为快捷方式
