# 权限系统架构文档（As-Is）

> 本文档描述当前**真实实现**，是 AI 修改权限代码前的第一参考。
> 设计规则与改动门禁见：`权限系统设计原则.md`

---

## 1. 目标与范围

- 目标：保证“管理员能力”和“用户能力”可并存，同时避免权限改动引发生产越权或误拒绝。
- 适用范围：
1. 后端 `auth/deps/rbac/activities/community/payments/users`
2. Web 权限分配与管理员页面
3. 小程序登录态、活动可见性、我的页、社区、支付
 

功能红线（部分）：
1、用户不能够没有被分配给它的活动类型的活动（公开除外） 
2、普通用户不能看到任何管理员的内容
3、未报名的用户不能看到活动的动态
---

## 2. 当前系统现状（必须按此理解）

### 2.1 身份不是互斥的（核心）

- 一个账号可以同时有两类能力：
1. 管理员能力（活动管理、用户管理、权限配置）
2. 普通用户能力（报名、支付、订单、评论）

- 这不是 bug，是产品要求。

### 2.2 Token 与权限分离

- JWT 里的 `role` 仅是粗粒度标记（`admin` / `user`）。
- 真正权限来自 DB 的 `UserRole` 实时查询（`crud_rbac`）。
- 因此：**不要把 `ctx.role` 当作最终权限结论**。

### 2.3 scope_type 是权限范围事实源

- `UserRole.scope_type` 取值：
1. `NULL`：全局
2. `activity_type`：活动类型范围
3. `activity`：单活动范围

- 判定全局权限必须看 `scope_type is NULL`，不能靠 `role_id` 猜。

### 2.4 权限判断是 OR 逻辑

- 用户有多个 `UserRole` 时：任一角色满足即可放行。
- 不存在“默认交集”语义。

### 2.5 活动可见性双视角

- 管理员视角：按 RBAC scope 过滤。
- 用户视角：按 `is_public` + `user_activity_types` 过滤。
- 小程序活动查询已引入 `as_user_view=1`，用于混合账号按用户规则看活动。

### 2.6 Web / 小程序用户列表接口分离

- `GET /users/admin/all`：保留脱敏（兼容旧链路/小程序）。
- `GET /users/admin/all-web`：Web 专用明文。
- 任何“只改 Web 不影响小程序”的用户列表需求，必须走 `all-web`。

## 3. 关键模块现状

### 3.1 Activities

- `activities` 列表、未开始列表、详情支持 `as_user_view`。
- `as_user_view=true` 时走用户可见性，不叠加管理员视角。

### 3.2 Payments

- 已移除“`ctx.role == admin` 一刀切拒绝”。
- 当前按“是否有有效登录用户上下文（`user_id/tenant_id`）”准入支付。

### 3.3 Community

- 读取：满足以下任一条件可读
1. 对该活动有管理权限（如 `participant.view` 或 `activity.edit` 的活动级判定）
2. 作为用户已报名该活动

- 评论：按“可读前提 + 登录身份”执行。

### 3.4 MiniProgram Auth

- `isAdmin()`：管理员能力。
- `isUser()`：普通用户能力，已兼容混合账号（有用户身份即可）。
- 管理员登录会保留 `user_id/user_name`，用于“我的活动/订单/头像昵称”等用户侧能力。

## 4. 已发生过的真实问题（避免你踩坑）

1. 分配活动类型后前端无生效：接口返回结构解析错误 + 未校验 `response.error`。
2. 小程序活动看不到：请求未带 token，后端按游客处理只返回公开活动。
3. 混合账号看不到分配活动：活动页前端又做了一次管理员二次过滤，把后端结果筛掉。
4. 管理员“我的”页显示“活动管理员”而非本人：登录态清空了用户信息或优先使用脱敏缓存名。
5. 管理员无法报名/支付/评论：后端用 `ctx.role == "admin"` 直接拒绝。
6. 社区查询报错：CRUD 用 `User.name.label(...)` 查询加密属性导致运行时异常。
7. 管理员看不到社区频道：`list_user_channels()` 只按成员表过滤，管理员未被加入成员表时返回空列表。已修复：管理员通过 `ctx.has_any_role(db)` 判定后直接返回租户下所有频道。
