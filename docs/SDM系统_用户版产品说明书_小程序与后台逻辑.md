# SDM 系统用户版产品说明书

> 适用范围：本文档面向使用 SDM 系统微信小程序的普通用户、活动管理员、社区管理员。  
> 本文档聚焦“小程序怎么用”以及“系统在后台会怎样处理”，暂不展开 Web 管理端说明。

---

## 1. 这套系统是做什么的

SDM 系统是一套围绕“活动”和“社区”展开的服务系统。  
用户可以在微信小程序里完成浏览活动、报名、支付、查看订单、加入社区、浏览公告、查看社区日历、管理个人资料等操作。  
对于有权限的管理员，还可以直接在小程序里发布活动、查看报名情况、处理签到、管理社区成员和审核内容。

从用户视角看，这套系统主要解决三件事：

1. 找到合适的活动，并顺畅完成报名。
2. 在活动之外，继续通过社区保持连接、接收通知和查看安排。
3. 让管理员也能在手机端完成日常运营工作。

## 2. 你在小程序里会看到什么

小程序当前主要由三部分组成：

1. `活动`：看活动、报名、支付、查看自己参加过的活动。
2. `社区`：加入社区后，看动态、公告、日历、通知。
3. `我的`：查看个人资料、订单、已报名活动、设置与协议说明。

如果你是管理员，系统还会根据你的权限，额外显示：

1. 发布活动
2. 报名管理
3. 签到管理
4. 统计查看
5. 用户查看与拉黑管理
6. 社区成员管理
7. 社区内容审核

## 3. 登录、注册与账号资料

### 3.1 登录方式

用户可以通过以下方式进入系统：

1. 手机号 + 密码登录
2. 微信快捷登录
3. 新用户注册后自动登录

如果系统发现你的登录状态已经失效，会直接把你带回登录页，而不是让你停留在一个已经不能继续操作的页面里。

### 3.2 首次使用时要做什么

首次进入时，系统可能会要求你补全个人资料，例如姓名、性别、年龄、行业、职业、手机号、邮箱等。  
这些信息主要用于活动报名、社区展示和管理员联系，不需要你每次报名都重新完整填写。

如果资料还没有补全，系统会在你报名活动前提醒你先完善资料，避免报名信息不完整。

1. 修改个人资料
2. 更换头像
3. 查看协议和说明
4. 注销账号

账号注销后，当前登录状态会失效，你需要重新登录或重新注册后才能继续使用。

## 4. 活动模块怎么用

### 4.1 浏览活动

活动首页会展示当前可浏览的活动内容。  
你可以查看活动名称、时间、地点、封面和基础说明，再进入详情页了解更多信息。

系统会优先把“可参与”的活动展示出来，通常包括：

1. 还未开始的活动
2. 正在进行中的活动

已经结束的活动一般不会作为主要报名入口展示给普通用户。

【截图占位-06：活动首页】  
建议放图：活动列表页。

### 4.2 查看活动详情

进入活动详情页后，你通常可以看到：

1. 活动封面
2. 活动名称
3. 开始和结束时间
4. 地点
5. 活动介绍
6. 是否需要报名
7. 是否需要支付

如果你有管理权限，这个页面还会额外出现活动编辑、报名管理、签到管理、统计查看等入口。

【截图占位-07：活动详情页】  
建议放图：普通用户视角和管理员视角各一张更好。

### 4.3 报名活动

报名时，系统会读取你的个人资料，并要求你补充一些活动问卷内容，例如：

1. 参与原因
2. 了解渠道
3. 学习期待
4. 对活动的理解
5. 是否还有问题

这样做的好处是，活动主办方能更快了解报名者情况，也能减少现场再重复沟通。

【截图占位-08：活动报名页】  
建议放图：报名表单页。

### 4.4 后台会怎样处理你的报名

这是用户最关心的一段逻辑，建议直接告诉用户：

1. 免费活动：提交成功后，系统会直接记录你的报名结果。
2. 付费活动：如果还有名额，系统会先拉起微信支付，支付成功后才算正式报名成功。
3. 已满活动：系统会把你放进候补名单，而不是直接失败。
4. 候补活动：如果后续有人退出或管理员处理名额，候补用户才有机会转为正式报名。

也就是说，**对付费活动来说，“下单了”不等于“报名成功了”，支付成功才算真正占到名额。**

建议放图：支付前、支付成功、候补提示三种状态。

### 4.5 支付、取消与订单

如果活动需要支付，系统会做这些处理：

1. 创建支付订单
2. 调起微信支付
3. 支付成功后自动回写报名状态
4. 如果你中途取消支付，系统会尽量把这笔待支付记录一起清掉，避免留下无效报名

在“我的订单”里，你可以查看自己的订单记录。  
在“我的活动”里，你可以看到自己是“已报名”“候补中”还是“报名处理中”。

【截图占位-10：我的订单 / 我的活动】

### 4.6 活动提醒

如果你同意了订阅消息，系统会在活动开始前发送提醒。  
当前规则是：系统会对已经审核通过、并且报名状态有效的用户发送活动开始前提醒。

从用户角度理解，可以把它看成：

1. 你报名成功了
2. 你授权了消息提醒
3. 活动快开始时，系统会主动提醒你

## 5. 社区模块怎么用

### 5.1 社区是什么

社区可以理解为“活动之外继续交流的地方”。  
加入某个社区后，你可以在里面：

1. 看社区动态
2. 发布内容
3. 评论互动
4. 查看公告
5. 查看社区日历
6. 接收社区邀请和提醒

【截图占位-12：社区首页 / 社区列表】  
建议放图：社区首页、频道卡片列表。

### 5.2 社区动态

进入社区后，最常见的是“动态流”。  
成员可以发布文字和图片内容，其他成员可以继续评论互动。

从用户体验上看，它更像一个围绕兴趣或活动主题展开的小型交流区。  
你可以把活动后的感受、照片、补充信息发到这里，也可以看别人发的内容。

【截图占位-13：社区动态列表】  
建议放图：动态流页面、评论展开页面。

### 5.3 社区内容在后台怎么处理

普通成员发布的内容，系统可能会进入待审核状态；管理员处理后，内容才会正式展示。  
这样做的目的是减少不合适内容直接出现在社区里。

但社区管理员发布的动态和公告，系统会直接通过，不再重复进入审核流程。  
因为这类内容通常本身就承担通知和组织作用，需要更快发出。

用更简单的话说：

1. 普通成员发内容，系统会更谨慎。
2. 社区管理员发内容，系统会更高效。

建议放图：发动态页、审核列表或待审核状态示意。

### 5.4 社区公告

公告和普通动态不是一回事。

公告更适合发布：

1. 时间调整
2. 集合说明
3. 重要通知
4. 社区规则

在产品里，公告会以独立入口存在，不会和普通动态混在一起。  
频道管理员可以发布公告，成员可以查看公告详情。

当前公告的特点是：

1. 公告是独立内容区
2. 只读展示，不开放评论
3. 频道管理员或发布人可以删除
4. 公告发出后，频道成员都可以看到

建议放图：频道顶部公告卡片、公告列表、公告详情。

### 5.5 社区日历

社区日历是一个非常适合做“安排同步”的模块。  
它可以用来展示：

1. 活动安排
2. 社区内部提醒
3. 某一天的重要事项
4. 需要成员提前准备的时间节点

用户进入日历后，可以按月查看，也可以点进某一天看当天事件。  
如果你是频道管理员，还可以新增、编辑和删除日历事件。

社区通知页主要承接两类信息：

1. 邀请消息
2. 站内提醒

例如别人邀请你加入某个社区时，你可以在通知页里直接接受或拒绝。  
通知也支持“全部已读”这类基础处理。

1. 创建活动
2. 编辑活动
3. 设置时间、地点、封面、报名人数
4. 设置是否收费
5. 设置活动类型

## 8. 系统在后台还做了哪些事情

这一章不讲技术实现细节，只讲“用户能感知到的后台规则”。

### 8.1 数据会按团队或租户隔离

如果系统服务于多个团队或多个组织，系统会尽量把数据分开处理。  
简单理解就是：你所在团队看到的活动、社区和数据，默认是和其他团队隔离的。

### 8.2 敏感信息不会直接裸露展示

系统会尽量保护敏感信息，例如手机号、邮箱等不会随意明文暴露给所有人。  
有权限的人可以在工作场景下查看必要信息，但系统本身会尽量减少无关暴露。

### 8.3 登录失效会自动拦截

当登录状态失效时，系统会优先让你重新登录，而不是继续让你在旧页面上误操作。  
这能减少“看起来点了，但其实没权限”的困扰。

### 8.4 失败时会给出明确提示

无论是报名失败、上传失败、支付失败，还是权限不足，系统都会尽量给出明确提示，而不是静默失败。  
这样用户至少知道发生了什么，也知道下一步该怎么做。

## 9. 建议给用户重点说明的几个规则

如果这份文档后续还要给更多用户看，建议把下面这几条单独高亮出来：

1. 付费活动必须支付成功，才算正式报名成功。
2. 满员后不一定是失败，也可能进入候补。
3. 报名前如果资料不完整，系统会要求先补资料。
4. 社区公告和社区动态是两种不同内容。
5. 管理员发布的公告和部分管理内容会直接生效，普通成员内容可能需要审核。
6. 被拉黑用户会被限制报名。
7. 如果登录失效，系统会要求重新登录。
