## 二、安全性问题

### 2.1 高危问题

#### 🔴 SEC-001: JWT Secret 默认值不安全
**文件**: `backend/app/core/config.py:15`
```python
JWT_SECRET: str = "change-me-in-production"
```
**风险**: 生产环境未配置环境变量时使用默认密钥，攻击者可伪造任意用户 Token。

**修复方案**:
1. 移除默认值，强制要求配置
2. 应用启动时检查是否为默认值，拒绝启动

---

#### 🔴 SEC-002: CORS 配置过于宽松
**文件**: `backend/app/main.py:19-25`
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
**风险**: 允许任意域名跨域访问，配合 `allow_credentials=True` 可能导致 CSRF 攻击窃取用户凭证。

**修复方案**:
1. 添加环境变量 `CORS_ORIGINS` 配置允许的域名列表
2. 生产环境禁止使用 `*` 配置

---

#### 🔴 SEC-003: 登录限流使用内存存储
**文件**: `backend/app/api/v1/endpoints/auth.py:22`
```python
_login_attempts: dict[str, list[float]] = defaultdict(list)
```
**风险**:
- 服务重启后限流记录丢失
- 无内存上限，可能被恶意 IP 耗尽内存导致 OOM
- 多实例部署时各实例独立限流（若部署多实例需额外处理）

**修复方案**:
1. **优化内存方案**（推荐，适用于单实例部署）:
   - 添加内存上限（如最多存储 10000 个 IP）
   - 定期清理过期记录（每次检查时清理）
   - 添加 IP 黑名单功能
2. **多实例部署替代方案**:
   - Nginx 层限流（推荐）：使用 `limit_req_zone` 和 `limit_req`
   - 数据库限流表：记录 IP 和时间戳，定时清理

---

#### 🔴 SEC-004: 微信 access_token 使用内存缓存

   ```python
   # 简单的内存缓存实现
   from functools import lru_cache
   from datetime import datetime, timedelta

   _cache: dict = {}
   _cache_expire: dict = {}

   def get_cached(key: str, ttl_seconds: int, loader):
       """通用缓存获取函数"""
       now = datetime.now()
       if key in _cache and _cache_expire.get(key, now) > now:
           return _cache[key]
       value = loader()
       _cache[key] = value
       _cache_expire[key] = now + timedelta(seconds=ttl_seconds)
       return value
   ```

**文件**: `backend/app/crud/crud_activity.py:164-188`

**问题代码**:
```python
def delete_activity(db: Session, activity_id: int, tenant_id: int):
```

```python

    try:
        db.query(ActivityParticipant).filter(...).delete()
        db.query(CheckInRecord).filter(...).delete()
        db.delete(db_activity)
        db.commit()
    except Exception as e:
        db.rollback()
        raise
```

## 八、应用内存缓存设计

### 缓存管理器实现

```python
# backend/app/utils/cache.py
from datetime import datetime, timedelta
from typing import Any, Callable, Optional
import threading

class SimpleCache:
    """简单的线程安全内存缓存"""

    def __init__(self):
        self._cache: dict[str, Any] = {}
        self._expire: dict[str, datetime] = {}
        self._lock = threading.Lock()

    def get(self, key: str) -> Optional[Any]:
        """获取缓存值，过期返回 None"""
        with self._lock:
            if key not in self._cache:
                return None
            if datetime.now() > self._expire.get(key, datetime.max):
                del self._cache[key]
                del self._expire[key]
                return None
            return self._cache[key]

    def set(self, key: str, value: Any, ttl_seconds: int):
        """设置缓存值"""
        with self._lock:
            self._cache[key] = value
            self._expire[key] = datetime.now() + timedelta(seconds=ttl_seconds)

    def delete(self, key: str):
        """删除缓存"""
        with self._lock:
            self._cache.pop(key, None)
            self._expire.pop(key, None)

    def clear(self):
        """清空所有缓存"""
        with self._lock:
            self._cache.clear()
            self._expire.clear()

# 全局缓存实例
cache = SimpleCache()

def cached(key: str, ttl_seconds: int, loader: Callable[[], Any]) -> Any:
    """缓存获取辅助函数"""
    value = cache.get(key)
    if value is not None:
        return value
    value = loader()
    cache.set(key, value, ttl_seconds)
    return value
```

### 使用示例

```python
# 缓存租户信息
def get_tenant_cached(db: Session, tenant_id: int) -> Tenant:
    return cached(
        f"tenant:{tenant_id}",
        ttl_seconds=600,  # 10 分钟
        loader=lambda: db.query(Tenant).filter(Tenant.id == tenant_id).first()
    )

# 缓存用户权限范围
def get_admin_scope_cached(db: Session, admin_id: int, tenant_id: int):
    return cached(
        f"admin_scope:{admin_id}:{tenant_id}",
        ttl_seconds=300,  # 5 分钟
        loader=lambda: crud_admin.get_admin_scope(db, admin_id, tenant_id)
    )
```

### 缓存键规范

| 数据类型 | 键格式 | TTL |
|---------|-------|-----|
| 租户信息 | `tenant:{tenant_id}` | 600s |
| 活动类型列表 | `activity_types:{tenant_id}` | 600s |
| 用户权限范围 | `admin_scope:{admin_id}:{tenant_id}` | 300s |
| 微信 access_token | `wechat_token:{appid}` | 5400s |

### 多实例部署说明

如果未来需要多实例部署，可选择以下方案：

1. **Nginx 负载均衡 + 会话保持**：同一用户请求始终路由到同一实例
2. **数据库缓存表**：将缓存数据存入数据库，适合更新频率低的场景
3. **独立缓存服务**：后续可独立部署缓存服务，无需改动业务代码
