# SDM 系统 — Web 端产品规格说明（Spec）

## 一、定位与目标

### 1.1 Web 端定位

Web 端是 SDM 系统的**主管理端**，承担平台运营、租户管理、权限配置、活动管理、用户治理与数据分析等桌面工作流。

与多端职责分工如下：

| 端 | 核心定位 |
|------|------|
| **Web** | 主管理端，负责复杂管理、配置、治理、统计 |
| **小程序** | 用户端 + 轻量管理端，负责活动浏览、报名、支付、分享、现场查看 |
| **Flutter App** | 用户端补充形态，可承接跨平台活动参与流程 |
| **Backend** | 多租户统一服务，提供鉴权、权限、业务与支付能力 |

### 1.2 Web 端目标

- 承载平台管理员与租户管理员的主要工作流
- 提供适合桌面场景的表格、筛选、分页、批量操作能力
- 明确多租户、角色、权限、可见范围与操作边界
- 逐步收敛当前散落在小程序中的后台能力
- 为后续报表、审计、导入导出、运营配置留足扩展空间

## 二、适用角色

### 2.1 角色分层

| 角色 | 定位 | 主要职责 |
|------|------|------|
| **平台管理员** | 跨租户平台运营 | 管理租户、平台账号、跨租户查看、平台级规则 |
| **租户超级管理员** | 单租户总负责人 | 管理本租户活动、用户、管理员、角色授权 |
| **活动类型管理员** | 单租户内的业务管理员 | 仅管理授权活动类型下的活动与报名 |
| **签到/现场人员** | 轻量运营角色 | 查看名单、现场签到、核销、看简单统计 |

### 2.2 使用原则

- **平台管理员入口只放 Web**
- **租户级后台能力以 Web 为主**
- **活动类型管理员可优先在 Web 完成配置型工作**
- **现场人员可在 Web 和小程序之间按场景分流**

## 三、信息架构

### 3.1 一级模块规划

建议 Web 端采用以下一级导航：

1. 工作台
2. 活动管理
3. 报名与签到
4. 用户管理
5. 权限与管理员
6. 租户管理
7. 数据报表
8. 系统设置

### 3.2 导航结构建议

| 一级模块 | 二级页面 | 说明 |
|------|------|------|
| **工作台** | 概览、待处理事项、近期活动、异常提醒 | 管理员首页 |
| **活动管理** | 活动列表、创建活动、编辑活动、活动详情、活动类型管理 | 核心运营模块 |
| **报名与签到** | 报名名单、签到记录、现场核销、候补管理 | 偏执行与现场 |
| **用户管理** | 用户列表、用户详情、黑名单、用户标签 | 用户治理 |
| **权限与管理员** | 管理员列表、角色管理、用户角色分配、权限查看 | RBAC 管理 |
| **租户管理** | 租户列表、租户详情、套餐与状态、租户管理员 | 平台管理员使用 |
| **数据报表** | 活动报表、报名转化、签到统计、支付统计 | 数据分析 |
| **系统设置** | 支付配置、上传配置、基础字典、审计日志 | 系统级配置 |

## 四、模块规格

### 4.1 工作台

#### 目标

帮助管理员在登录后快速了解当前租户或平台的运营状态。

#### 页面内容

- 核心指标卡片
- 今日待开始活动
- 待处理事项
- 最近报名与支付异常
- 快捷入口

#### 关键指标

- 活动总数
- 进行中活动数
- 今日报名人数
- 今日签到人数
- 待支付订单数
- 候补人数

#### 当前状态

- **已实现第一阶段骨架（2026-04-16）**
- 已新增正式后台导航结构与登录页
- 已落地工作台首页（核心指标、近期活动、待处理事项）
- 已重构活动管理列表，补充状态筛选、分页、状态流转、详情入口
- 已新增活动详情聚合页
- 已重构用户管理列表，补充搜索、分页、拉黑/解除拉黑、详情查看

---

### 4.2 活动管理

#### 目标

承接活动从创建、编辑、发布、状态变更到结束的全生命周期管理。

#### 页面

| 页面 | 说明 |
|------|------|
| **活动列表** | 表格展示活动，支持筛选、分页、排序、批量操作 |
| **创建活动** | 完整活动创建流程 |
| **编辑活动** | 编辑基础信息、时间、海报、费用、名额 |
| **活动详情** | 查看活动信息、报名情况、签到情况、支付情况 |
| **活动类型管理** | 管理活动类型，为权限范围提供基础数据 |

#### 活动列表字段建议

- 活动名称
- 活动类型
- 活动状态
- 开始时间 / 结束时间
- 地点
- 报名人数 / 名额
- 候补人数
- 是否付费
- 创建人
- 创建时间

#### 主要操作

- 新建活动
- 编辑活动
- 删除活动
- 修改状态
- 查看报名
- 查看签到
- 查看统计

#### 权限要求

- 平台管理员：可跨租户查看
- 租户超级管理员：可管理本租户全部活动
- 活动类型管理员：仅可操作授权类型下活动

#### 当前状态

- **部分实现**
- 现有 Web 已有活动列表、创建活动、编辑活动、查看参与人
- 本轮已补齐筛选、分页、活动详情聚合页、状态快捷流转
- 仍缺批量操作、活动类型管理、更完整的导出能力

---

### 4.3 报名与签到

#### 目标

把报名、签到、候补、现场核销能力从“活动详情附属能力”提升为独立运营模块。

#### 页面

| 页面 | 说明 |
|------|------|
| **报名名单** | 按活动查看报名用户、候补用户、支付状态 |
| **签到记录** | 查看签到时间、签到人、活动归属 |
| **现场核销** | 面向桌面现场或大屏配套使用 |
| **候补管理** | 查看候补、手动转正、通知跟进 |

#### 报名名单能力

- 按活动筛选
- 按报名状态筛选
- 按支付状态筛选
- 搜索用户姓名 / 手机 / 证件号
- 导出报名名单

#### 签到管理能力

- 查看签到记录
- 按活动筛选
- 手动签到
- 核销异常提示

#### 当前状态

- **部分实现**
- Web 已有签到页和活动参与人页
- 本轮已将“报名名单”统一纳入活动详情与活动列表入口
- 仍缺候补管理、报名状态筛选、支付状态联动、导出能力、独立签到记录中心

### 4.5 权限与管理员

#### 目标

把后端已有的 RBAC 能力做成可视化管理端。

#### 页面

| 页面 | 说明 |
|------|------|
| **管理员列表** | 查看租户内管理员与角色 |
| **角色管理** | 查看角色、角色权限、系统角色 / 自定义角色 |

#### 关键能力

- 查看所有角色

- 后端已有 `roles` 相关接口与 `role.manage`、`admin.manage` 权限模型

## 五、权限边界

### 5.1 平台管理员

- 可访问租户管理
- 可跨租户查看用户与活动数据
- 可查看平台级报表
- 不直接参与普通活动执行流程

### 5.2 租户超级管理员

- 可管理本租户全部活动、用户、管理员与角色
- 不可跨租户查看数据
- 是租户内完整后台的拥有者

### 5.3 活动类型管理员

- 只能查看和管理授权活动类型内的活动
- 可查看对应活动报名与签到
- 不可管理租户级用户、角色、租户信息

### 5.4 现场执行人员

- 重点使用签到、核销、名单查看等能力
- 可在 Web 或小程序使用轻量功能
- 不承担复杂配置工作

## 六、与小程序的职责边界

### 6.1 Web 负责

- 平台管理员入口
- 租户管理
- 角色与权限配置
- 活动配置与完整编辑
- 用户治理与黑名单
- 数据报表
- 导入导出
- 审计日志

### 6.2 小程序负责

- 活动浏览
- 活动详情
- 报名
- 支付
- 分享承接
- 现场查看名单
- 简单状态修改
- 轻量签到查看

### 6.3 当前迁移建议

当前小程序已具备部分后台能力，但短期内可先保持现状。Web 端后续应逐步接手以下能力：

1. 用户管理
2. 黑名单管理
3. 创建活动与编辑活动的主入口
4. 权限配置
5. 租户管理

## 九、实施建议

### 9.1 第一阶段

先把 Web 端从“零散工具页”升级为“正式后台框架”。

建议优先完成：

- 登录后工作台框架
- 新侧边栏导航
- 活动列表重构
- 活动详情页
- 用户列表重构

第一阶段实际落地顺序：

1. 登录页与后台主框架
2. 工作台首页
3. 活动管理列表与详情页
4. 用户管理重构
5. 其余模块先预留正式入口页

### 9.2 第二阶段

在活动与用户主流程稳定后，补权限与租户能力。

建议补齐：

- 角色管理
- 管理员分配
- 活动类型管理
- 租户管理

### 9.3 第三阶段

最后补足平台运营能力。

建议补齐：

- 数据报表
- 导入导出
- 审计日志
- 异常提醒
