# 小程序 UI 留白与间距规范

## 1. 目的

这份规范用于统一 SDM 小程序的内页视觉节奏，重点约束：

- 页面边距
- 模块之间的留白
- 卡片/表单/信息块的组织方式
- 标题、辅助文本、状态文案的层级
- 空态、加载态、错误态的呈现方式

目标不是做“装饰感很强”的页面，而是形成一种稳定、克制、可复用的阅读节奏。

## 2. 参考页

以下页面作为当前最稳定的视觉参考：

- `pages/activity-list/activity-list`
- `pages/activity-detail/activity-detail`
- `pages/register/register`

它们共同的特点是：

- 纵向单列结构
- 低视觉噪音
- 充足留白
- 少阴影、少重边框
- 信息按层级自然展开

## 3. 核心原则

### 3.1 先结构，后装饰

页面优先保证内容结构清晰，再考虑视觉装饰。

推荐结构：

1. `page-header`
2. 主内容区
3. 底部操作区

### 3.2 留白要有节奏

留白不是越多越好，而是要形成层次：

- 页面左右边距保持统一
- 模块间距明显大于模块内部间距
- 同类元素之间保持一致节奏
- 主操作区与内容区要分离

### 3.3 视觉表达要克制

优先使用：

- 白底或极浅底色
- 轻分割线
- 轻圆角
- 少量强调色

避免：

- 大面积厚重色块
- 过强阴影
- 过多边框叠加
- 复杂装饰性背景

### 3.4 信息层级要清晰

通常遵循以下顺序：

1. 页面标题
2. 主要模块标题
3. 主要内容
4. 辅助说明
5. 状态提示

标题和正文要靠字号、字重、颜色和间距区分，不靠花哨装饰区分。

## 4. 间距建议

以下是当前页面中比较稳定的一组节奏，可作为默认值：

- 页面左右内边距：`32rpx` 左右
- 页头下方首个模块起始间距：`28rpx ~ 76rpx`
- 模块之间间距：`28rpx ~ 68rpx`
- 模块内部条目间距：`10rpx ~ 24rpx`
- 卡片内边距：`24rpx ~ 32rpx`
- 底部按钮区内边距：`24rpx ~ 34rpx`

推荐的判断标准：

- 如果内容是“同一组信息”，就用小间距连接
- 如果内容是“不同语义模块”，就用更大的间距分开

## 5. 组件与容器规范

### 5.1 页面容器

- 页面主体使用白底或极浅背景
- 页面内容以单列流式布局为主
- 不要在同一页混用太多不同风格的容器

### 5.2 卡片

- 卡片尽量只承担一个语义块
- 卡片圆角保持统一
- 阴影只在必要时使用，且必须很轻
- 卡片内边距要足够，避免内容贴边

### 5.3 表单

- 表单项尽量统一为“标题 + 输入区”结构
- 输入区优先用轻边框、下划线或极简分隔
- 同一页面内不要同时出现太多不同风格的输入控件

### 5.4 状态区

- 加载态、空态、错误态都尽量居中
- 文案要短，语气要温和
- 不要让状态区比正文还重

## 6. 推荐写法

### 适合

- 单列卡片流
- 分段阅读页
- 极简表单页
- 详情信息页
- 状态反馈页

### 不适合

- 多栏复杂拼版
- 强装饰风格页面
- 过多密集控件堆叠
- 同页同时出现多种视觉系统

## 7. 页面自检清单

页面完成后，建议按下面顺序检查：

1. 是否使用了统一的顶部结构
2. 左右边距是否一致
3. 模块之间是否有明确呼吸感
4. 是否避免了过重阴影和边框
5. 标题、正文、辅助文字是否清晰分层
6. 底部操作是否与内容区分离
7. 空态/加载态/错误态是否足够克制

## 8. 一句话总结

这套小程序 UI 的核心不是“装饰”，而是“秩序感”。  
只要页面能做到单列、克制、留白足、层级清楚，基本就会自然贴近这套风格。
