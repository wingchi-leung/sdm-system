# 鉴权说明

## 一、双角色体系概览

系统有两种登录身份，各用各的表、各走各的登录接口，但共用同一套 JWT 机制。

| 角色 | 数据表 | 登录接口 | 登录凭证 | JWT 中的 role |
|------|--------|----------|----------|---------------|
| **管理员** | `admin_user` | `POST /api/v1/auth/login` | 用户名 + 密码 | `"admin"` |
| **普通用户** | `user` | `POST /api/v1/auth/user-login` | 手机号 + 密码 | `"user"` |
| **普通用户（微信）** | `user`（按 wx_openid 匹配） | `POST /api/v1/auth/wechat-login` | 小程序 wx.login() 的 code | `"user"` |

---

## 二、两张表各干什么

### admin_user（管理员表）

```
id | username | password_hash | user_id(可选)
```

- 用于**管理端登录**（发布活动、签到管理等）
- `password_hash` 必填，bcrypt 哈希
- `user_id` 可选，用于关联 user 表（标识这个管理员对应哪个用户）

### user（用户表）

```
id | name | phone(唯一) | password_hash | email | isblock | block_reason | wx_openid(唯一,可选) | ...
```

- 用于**普通用户注册/登录**（报名活动、查看个人信息等）
- `phone` 有唯一索引，作为登录凭证；微信登录用户占位为 `wx_{openid}`
- `password_hash` 注册时写入，bcrypt 哈希；微信登录用户可为空
- `wx_openid` 微信小程序 openid，唯一索引，用于 `POST /api/v1/auth/wechat-login` 自动注册/登录
- `isblock=1` 表示黑名单，登录和报名时会被拦截

## 三、JWT Token 机制

### 生成

两种角色登录成功后都调用同一个函数生成 token：

```python
# app/core/security.py
def create_access_token(sub: str, role: str = "admin") -> str:
    payload = {"sub": str(sub), "role": role, "exp": expire}
    return jwt.encode(payload, JWT_SECRET, algorithm="HS256")
```

- 管理员：`sub = admin_user.id`，`role = "admin"`
- 普通用户：`sub = user.id`，`role = "user"`

### Token 有效期

默认 7 天（`JWT_EXPIRE_MINUTES = 60 * 24 * 7`），在 `.env` 或 `config.py` 中配置。

### 客户端使用

所有需要鉴权的请求，在 Header 中携带：

```
Authorization: Bearer <token>
```

## 四、三种鉴权依赖（后端使用）

在 `app/api/deps.py` 中定义了三个 FastAPI 依赖项，按需选用：

| 依赖函数 | 用途 | 返回值 |
|----------|------|--------|
| `get_current_admin` | **仅管理员**可访问 | `int`（admin_user.id） |
| `get_current_user` | **任意已登录用户**可访问 | `{"id": int, "role": str}` |
| `get_current_user_optional` | **可选登录**，不登录也能访问 | `{"id": int, "role": str}` 或 `None` |

### 使用示例

```python
# 仅管理员可创建活动
@router.post("/")
def create_activity(admin_id: int = Depends(get_current_admin)):
    ...

# 任意已登录用户可查看自己的信息
@router.get("/me")
def get_my_profile(current: dict = Depends(get_current_user)):
    if current["role"] != "user":
        raise HTTPException(403, "仅限普通用户")
    user = get_user(db, current["id"])
    ...

# 报名接口：不强制登录，但有 token 就解析
@router.post("/")
def create_participant(current: dict | None = Depends(get_current_user_optional)):
    ...
```

## 五、安全措施

### 登录限流

每个 IP 在 60 秒内最多尝试 10 次登录（管理员和用户共用），超出返回 `429`。

配置项（`config.py`）：
- `LOGIN_RATE_LIMIT_COUNT: int = 10`
- `LOGIN_RATE_LIMIT_WINDOW_SECONDS: int = 60`

### HTTPS 强制（可选）

设置 `REQUIRE_HTTPS_FOR_LOGIN = True` 后，登录接口会拒绝非 HTTPS 请求。开发环境保持 `False`。

### 密码长度限制

bcrypt 只支持最多 72 字节的密码，登录时会先校验，超出直接返回 400。

### 黑名单拦截

- **用户登录时**：检查 `user.isblock`，拉黑用户直接返回 `403`
- **报名时**：按手机号查 user 表，若 `isblock=1` 返回 `403`

## 六、各接口的鉴权要求

| 接口 | 鉴权要求 |
|------|----------|
| `POST /auth/login` | 无（管理员登录入口） |
| `POST /auth/user-login` | 无（用户登录入口） |
| `POST /users/register` | 无（用户注册入口） |
| `GET /users/me` | 需要 user token |
| `POST /activities/` | 需要 admin token |
| `POST /participants/` | 可选 token（有则用于黑名单拦截） |
| `GET /activities/`、`GET /activities/unstarted/` | 无 |
| `GET /users/`、`GET /users/{id}` | 无 |
| `POST /checkins/` | 无 |

## 七、首次部署：创建管理员

### 1. 生成密码哈希

在 `backend` 目录下执行：

```bash
python scripts/hash_admin_password.py 你的密码
```

会输出一长串 bcrypt 哈希，例如 `$2b$12$xxxx...`。

### 2. 插入管理员

在 MySQL 中执行（把 `<hash>` 替换为上一步的输出）：

```sql
INSERT INTO admin_user (username, password_hash)
VALUES ('admin', '<hash>');
```

### 3. 验证

```bash
python scripts/verify_admin_password.py admin 你的密码
```

输出「校验通过」即可。

## 八、数据库迁移（如果是从旧版升级）

如果你的 user 表还没有 `password_hash` 列和 `phone` 唯一索引，需要执行：

```sql
ALTER TABLE `user` ADD COLUMN `password_hash` varchar(255) DEFAULT NULL COMMENT '用户密码哈希' AFTER `email`;
ALTER TABLE `user` ADD UNIQUE INDEX `uk_user_phone` (`phone`);
```

执行后重启后端即可。

## 九、常见问题

### 管理员登录报「用户名或密码错误」

1. 确认用的是 `admin_user` 表，不是 `user` 表
2. 运行 `python scripts/verify_admin_password.py admin 你的密码` 校验
3. 如果校验失败，重新生成哈希并 `UPDATE admin_user SET password_hash='...' WHERE username='admin'`
4. 确认用户名大小写完全一致

### 普通用户登录报「手机号或密码错误」

1. 确认该手机号已通过 `POST /users/register` 注册过
2. 确认 user 表中该手机号对应的 `password_hash` 不为 NULL
3. 确认密码拼写正确

### 登录报 429

登录尝试过于频繁，等 60 秒后重试。

### 登录报 403「账号已被拉黑」

该用户的 `isblock` 被设为了 1，需要管理员在数据库中修改：

```sql
UPDATE user SET isblock = 0, block_reason = NULL WHERE phone = '手机号';
```
